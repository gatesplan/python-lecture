## 2번 (코드 완성)

아래 코드는 리스트 `arr`에서 값 `x`를 **순차 탐색**으로 찾아 해당 인덱스를 반환하는 프로그램이다. 값을 찾지 못하면 `-1`을 반환하도록 한다. `search` 함수의 본문을 완성하시오.

```python
arr = [4, 6, 10, 2, 3, 1, 7, 5]

def search(x):
    # 이 부분을 완성하시오


print(search(7))
```

**정답 예시**:

```python
arr = [4, 6, 10, 2, 3, 1, 7, 5]

def search(x):
    for i in range(len(arr)):
        if arr[i] == x:
            return i
    return -1


print(search(7))
```

**해설**: 함수 바깥에서 정의된 `arr`을 함수 내부에서 **읽기만** 할 때는 `global` 선언이 필수는 아니지만, 외부 리스트를 참조한다는 점을 명시하려면 함수 첫 줄에 `global arr`을 쓸 수도 있다. 순차 탐색은 `for`문으로 인덱스 0부터 끝까지 하나씩 비교하며 일치하는 값을 찾은 즉시 해당 인덱스를 반환하고, 끝까지 찾지 못하면 반복문 종료 후 `-1`을 반환한다. 예시 출력은 `6`(값 7의 인덱스).

## 3번 (서술형)

**이진 탐색(Binary Search)**에 대해 아는 대로 서술하시오.

**모범 답안 (채점 포인트)**:

1. **정의**: 정렬된 리스트의 중간값을 확인하여, 찾으려는 값이 중간값보다 작으면 왼쪽 절반, 크면 오른쪽 절반만 탐색 대상으로 남기는 과정을 반복하며 값을 찾는 알고리즘이다.

2. **전제 조건**: 데이터가 반드시 **오름차순(또는 내림차순)으로 정렬**되어 있어야 한다. 정렬되지 않은 리스트에는 적용할 수 없다.

3. **작동 원리**: `left`, `right`, `mid` 세 개의 포인터를 사용한다.
   - `mid = (left + right) // 2`로 중앙 인덱스를 구한다(정수여야 하므로 `//` 사용).
   - `arr[mid] == target`이면 탐색 성공, `mid` 반환.
   - `arr[mid] < target`이면 `left = mid + 1`로 오른쪽 절반만 탐색.
   - `arr[mid] > target`이면 `right = mid - 1`로 왼쪽 절반만 탐색.
   - `left > right`가 되면 값이 없다는 뜻이므로 탐색 종료(실패, `-1` 반환).

4. **효율성**: 매 단계마다 탐색 범위를 절반씩 줄이기 때문에, 데이터가 n개일 때 **최악의 경우 약 log2(n)번**의 비교로 탐색이 끝난다. 데이터가 1,000,000개면 약 20번, 10,000,000개여도 약 24번 만에 찾을 수 있어 순차 탐색의 O(n)에 비해 압도적으로 빠르다.

5. **구현 방식**: `while`문을 사용하는 **반복(Iterative) 방식**과, 자기 자신을 호출하는 **재귀(Recursive) 방식** 두 가지로 구현할 수 있다. 반복 방식은 콜 스택을 사용하지 않아 메모리 효율이 좋고, 재귀 방식은 분할 정복의 논리를 코드에 그대로 매핑하여 가독성이 좋다.

6. **일상 예시**: 사전에서 단어 찾기(알파벳 순서로 정렬되어 있어 중간을 펼쳐보고 절반씩 좁혀가는 방식)가 대표적인 이진 탐색이다.

## 4번 (빈칸 채우기)

아래는 정렬된 리스트에서 특정 값을 찾는 **재귀적 이진 탐색** 코드이다. 빈칸 **(가), (나), (다)** 에 들어갈 내용을 각각 쓰시오.

```python
def binary_search(arr, target, low, high):
    if low > high:
        return -1
    mid = (low + high) // 2

    if arr[mid] == target:
        return mid
    elif (가) :
        return (나)
    else:
        return (다)


data = [2, 5, 6, 8, 9, 11, 15, 18, 19, 22, 24, 25]
result = binary_search(data, 15, 0, len(data) - 1)
print(result)
```

**정답**:

- (가): `arr[mid] > target`
- (나): `binary_search(arr, target, low, mid - 1)`
- (다): `binary_search(arr, target, mid + 1, high)`

**해설**:

- `arr[mid] == target`에서 걸리지 않았다는 것은 `arr[mid]`와 `target`이 서로 다르다는 뜻이다. 이 경우는 "중간값이 타겟보다 크다" 또는 "중간값이 타겟보다 작다" 둘 중 하나이므로, (가)에는 둘 중 하나의 조건을 적는다. 정답 예시는 `arr[mid] > target`(중간값이 타겟보다 큰 경우).
- (가)가 참이면 찾는 값은 **왼쪽 절반**에 있으므로, `high`를 `mid - 1`로 좁혀서 `binary_search` 자기 자신을 다시 호출한다 → (나).
- (가)가 거짓이면 `arr[mid] < target`이라는 뜻이므로 찾는 값은 **오른쪽 절반**에 있다. `low`를 `mid + 1`로 좁혀서 재귀 호출한다 → (다).
- (가) 조건을 `arr[mid] < target`으로 쓴다면 (나), (다)의 재귀 호출 범위도 서로 바뀌어야 한다(오른쪽 절반 ↔ 왼쪽 절반).

## 5번 (오류 분석 + 코드 수정)

아래 코드는 학생 명단에서 `"이효주"` 학생의 위치(인덱스)를 **이진 탐색**으로 찾으려는 프로그램이다. 그러나 실제로 실행해보면 명단에 `"이효주"`가 분명히 있음에도 `-1`이 반환되어 "찾지 못했다"는 결과가 나온다.

```python
names = ["박민서", "강수빈", "이효주", "김서연", "윤예서",
         "최지민", "한윤서", "김지우", "이서준", "김하은"]


def binary_search(target, low, high):
    if low > high:
        return -1
    mid = (low + high) // 2
    if names[mid] == target:
        return mid
    elif names[mid] > target:
        return binary_search(target, low, mid - 1)
    else:
        return binary_search(target, mid + 1, high)


result = binary_search("이효주", 0, len(names) - 1)
print(result)
```

**문제**
(1) 왜 `"이효주"`를 찾지 못하는지, 이진 탐색 알고리즘의 원리에 근거하여 설명하시오.
(2) 이 문제를 해결하기 위해 **적절한 위치에 코드 한 줄**을 추가하시오. 추가할 코드와 추가할 위치(어느 줄 앞/뒤인지)를 함께 적으시오.

---

**정답**

**(1) 원인**

이진 탐색은 리스트의 중간값과 타겟을 비교해서 "타겟이 중간값보다 크면 오른쪽 절반, 작으면 왼쪽 절반"으로 탐색 범위를 좁혀가는 알고리즘이다. 이 "절반을 버린다"는 판단이 성립하려면 리스트가 **사전순(또는 크기순)으로 정렬되어 있어야 한다**는 절대 전제 조건이 있다.

그런데 `names` 리스트는 정렬되어 있지 않고 순서가 뒤섞여 있다. 예를 들어 첫 재귀 호출에서 중간 인덱스(`mid = 4`)의 값은 `"윤예서"`인데, `"이효주" > "윤예서"`이므로 알고리즘은 오른쪽 절반(인덱스 5~9)으로 탐색을 이어간다. 하지만 실제 `"이효주"`는 왼쪽 절반(인덱스 2)에 있다. 정렬 전제가 깨졌기 때문에 "절반을 버리는" 판단 자체가 틀린 방향을 가리키게 되고, 결국 타겟을 건너뛴 채 `low > high` 조건에 걸려 `-1`이 반환된다.

**(2) 수정**

`binary_search`를 호출하기 **직전**에 다음 한 줄을 추가한다.

```python
names.sort()
```

추가 위치: `result = binary_search("이효주", 0, len(names) - 1)` 줄 **바로 앞**.

수정 후 해당 부분:

```python
names.sort()
result = binary_search("이효주", 0, len(names) - 1)
print(result)
```

이렇게 하면 파이썬 문자열 비교가 유니코드 코드포인트 순서를 따르고, 이 순서는 현대 한글 사전순(가나다 순)과 일치하므로 이름 리스트가 사전순으로 정렬된다. 그 후 이진 탐색이 올바른 전제 위에서 동작하여 `"이효주"`의 인덱스를 정상적으로 반환한다.

## 6번 (코드 작성)

아래는 정수 리스트 `arr = [3, 1, 4, 2]`를 어떤 정렬 함수에 통과시켰을 때, **두 원소가 서로 자리를 바꿀 때마다** 그 직후의 리스트 상태를 한 줄씩 출력한 결과이다.

```
[3, 1, 4, 2]
[1, 3, 4, 2]
[1, 3, 2, 4]
[1, 2, 3, 4]
```

(첫 줄은 정렬 전 초기 상태, 이후 각 줄은 한 번의 자리 교환 직후의 상태이다.)

위 자리 교환 순서와 동일한 순서로 동작하여 리스트를 오름차순으로 정렬하는 함수 `sort(arr)`를 작성하시오. 함수는 정렬된 리스트를 반환해야 한다.

**정답 예시**:

```python
def sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-1-i):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr


print(sort([3, 1, 4, 2]))
```

**해설**:

각 줄의 변화를 따라가면 **인접한 두 원소가 서로 자리를 바꾸는 동작**이 반복되고 있음을 확인할 수 있다.

- 1→2번째 줄: 인덱스 0, 1의 교환 (3, 1 → 1, 3)
- 2→3번째 줄: 인덱스 2, 3의 교환 (4, 2 → 2, 4)
- 3→4번째 줄: 인덱스 1, 2의 교환 (3, 2 → 2, 3)

리스트를 앞에서부터 훑으면서 인접한 두 원소를 비교해 큰 쪽을 뒤로 보내는 과정을 반복하는 정렬이 **버블 정렬(Bubble Sort)** 이다. 한 번의 패스(`j`가 처음부터 끝까지 도는 동안)가 끝나면 그 패스에서 가장 큰 값이 맨 뒤 자리에 확정된다. 위 트레이스에서도 첫 번째 패스가 끝났을 때 가장 큰 값 `4`가 마지막 자리에 들어왔고(2→3번째 줄), 두 번째 패스가 끝났을 때 두 번째로 큰 값 `3`이 끝에서 두 번째 자리에 들어왔다(3→4번째 줄). 매 패스마다 뒤쪽 한 자리씩 정렬이 확정되므로 안쪽 반복문의 범위를 한 칸씩 줄여 `range(0, n-1-i)`로 쓴다.

## 7번 (서술형)

다음 코드를 실행하면 `[1, 99]`가 출력된다. 왜 이런 결과가 나오는지 아는 대로 서술하시오.

```python
a = [1, 2]
b = a
a[1] = 99
print(b)
```

**모범 답안 (채점 포인트)**:

1. **변수와 객체의 관계**: 파이썬에서 변수는 "값을 담는 상자"가 아니라 **객체에 붙은 이름표**이다. 대입 연산은 값을 복사하는 것이 아니라 이름을 특정 객체에 연결하는 것이다.

2. **`b = a`의 의미**: 이 대입은 `a`가 가리키는 리스트를 **복사하지 않는다**. `a`와 `b`는 메모리상 **하나의 같은 리스트 객체**에 붙은 두 개의 이름이 된다.

3. **`a[1] = 99`의 의미**: 이 연산은 `a`가 가리키는 **리스트 객체의 내부를 수정**한다(mutation). 리스트는 mutable 객체라서 이런 내부 변경이 가능하다. 새로운 리스트가 만들어지는 것이 아니라, 기존 리스트 객체 자체가 바뀐다.

4. **출력의 원인**: 3의 변경은 객체 자체에 가해진 것이고, `b`는 바로 그 객체를 가리키고 있다. 따라서 `b`를 직접 수정한 적이 없어도 `print(b)`에는 변경된 결과 `[1, 99]`가 그대로 보인다. 이는 두 이름이 같은 객체를 공유하고 있어서 생기는 현상이다.

## 8번 (서술형)

다음 코드를 실행하면 `[3, 4]`와 `[1, 2]`가 순서대로 출력된다. 7번과 비슷해 보이지만 `b`의 출력 결과가 다르다. 왜 이런 결과가 나오는지 아는 대로 서술하시오.

```python
a = [1, 2]
b = a
a = [3, 4]
print(a)
print(b)
```

**모범 답안 (채점 포인트)**:

1. **두 종류의 연산 구분**: `a[i] = ...`는 `a`가 가리키는 **객체의 내부**를 수정하는 연산(mutation)이고, `a = ...`는 `a`라는 **이름이 가리키는 대상 자체를 교체**하는 연산(rebinding)이다. 7번은 전자, 8번은 후자에 해당한다.

2. **`a = [3, 4]`의 의미**: 새로운 리스트 객체 `[3, 4]`가 메모리에 만들어지고, `a`라는 이름표가 이 새 객체로 옮겨간다. 이 과정에서 원래의 `[1, 2]` 객체는 내용이 바뀌지 않고 메모리에 그대로 남아 있다.

3. **`b`가 가리키는 대상**: `b = a`는 그 시점에 `a`가 가리키던 객체(`[1, 2]`)를 `b`도 함께 가리키게 한 것이다. 이후 `a`의 이름표가 다른 객체로 옮겨지더라도, `b`의 이름표는 원래 객체(`[1, 2]`)에 그대로 붙어 있다.

4. **출력의 원인**: `print(a)`는 새 객체 `[3, 4]`를, `print(b)`는 원래 객체 `[1, 2]`를 출력한다.

5. **7번과의 대조**: 겉보기에 비슷한 `a[1] = 99`(7번)와 `a = [3, 4]`(8번)가 완전히 다른 동작을 한다는 점이 핵심이다. 전자는 공유된 객체 자체를 바꾸므로 다른 이름(`b`)에도 그 변경이 보이지만, 후자는 `a`의 이름표만 새 객체로 옮기는 것이므로 원래 객체에 대한 `b`의 참조는 그대로 유지된다.

## 9번 (오류 분석 + 코드 수정)

아래 코드는 3×3 크기의 정수 격자 `grid`를 모두 0으로 초기화한 뒤, 좌측 상단 칸 `[0][0]`의 값만 1로 바꾸려는 프로그램이다.

```python
grid = [[0] * 3] * 3
grid[0][0] = 1
print(grid)
```

작성자의 의도대로라면 출력은 다음과 같아야 한다.

```
[[1, 0, 0], [0, 0, 0], [0, 0, 0]]
```

그러나 실제 실행하면 세 행이 모두 동시에 바뀌어 아래와 같이 출력된다.

```
[[1, 0, 0], [1, 0, 0], [1, 0, 0]]
```

**문제**
(1) 왜 세 행이 전부 동시에 바뀌는지, 객체와 참조 개념에 근거하여 설명하시오.
(2) 이 문제를 해결하기 위해 `grid = [[0] * 3] * 3` 한 줄을 어떻게 고쳐야 하는지 작성하시오.

---

**정답**

**(1) 원인**

`[0] * 3`은 `[0, 0, 0]`이라는 리스트 객체 **하나**를 메모리에 만든다. 그 다음 바깥의 `* 3`은 이 리스트 객체를 **내부적으로 세 번 반복**하는데, 이때 새 리스트 세 개가 만들어지는 것이 아니라 **이미 만들어진 단 하나의 리스트 객체를 가리키는 참조(이름표) 세 개**가 바깥 리스트에 담긴다.

즉 `grid[0]`, `grid[1]`, `grid[2]`는 모두 **메모리상 같은 하나의 리스트 객체**를 가리키는 서로 다른 이름이다. 따라서 `grid[0][0] = 1`은 "0번 행의 0번 칸을 바꾸는 것"이 아니라 "세 이름표가 공유하고 있는 하나의 리스트 객체의 0번 칸을 바꾸는 것"이며, 그 결과 `grid[1][0]`과 `grid[2][0]`으로 같은 객체를 조회해도 동일한 변경이 반영되어 세 행이 모두 `[1, 0, 0]`으로 보이게 된다. 이는 7번에서 `b = a` 후 `a[1] = 99`를 했을 때 `b`도 함께 바뀌던 현상과 같은 원리이다.

**(2) 수정**

```python
grid = [[0] * 3 for _ in range(3)]
```

**이유**: `for _ in range(3)`에 의해 반복이 돌 때마다 `[0] * 3`이 **매번 새로 평가되어 새 리스트 객체가 생성**된다. 그 결과 바깥 리스트에는 서로 다른 세 개의 리스트 객체가 담기게 되어, `grid[0][0] = 1`을 수행해도 다른 두 행에는 영향이 없다.

## 10번 (코드 작성)

아래는 어느 분식집의 3일치 메뉴별 판매량을 2차원 리스트로 정리한 표이다. 각 행은 하루치 판매 기록이고, 각 열은 왼쪽부터 순서대로 `[떡볶이, 라면, 순대, 튀김]`의 판매량을 나타낸다.

```python
sales = [
    [12, 8, 5, 10],   # 1일차
    [15, 6, 7, 12],   # 2일차
    [9, 11, 4, 8],    # 3일차
]
```

**메뉴별 3일간의 총 판매량**을 담은 리스트를 반환하는 함수 `total_by_menu(sales)`를 작성하시오. 위 예시의 경우 `[36, 25, 16, 30]`이 반환되어야 한다.

**정답 예시**:

```python
def total_by_menu(sales):
    menu_count = len(sales[0])
    totals = [0] * menu_count
    for row in sales:
        for j in range(menu_count):
            totals[j] += row[j]
    return totals


sales = [
    [12, 8, 5, 10],
    [15, 6, 7, 12],
    [9, 11, 4, 8],
]
print(total_by_menu(sales))   # [36, 25, 16, 30]
```

**해설**:

2차원 리스트에서 "행별 합계"는 바깥 루프로 행을 돌면서 안쪽에서 그 행의 합을 구하면 되지만, **열별 합계**는 조금 다른 사고가 필요하다. 같은 열 인덱스 `j`에 대해 **모든 행의 `j`번째 값을 누적**해야 하기 때문이다.

이를 위해 먼저 열 개수(=메뉴 개수)만큼 0으로 채운 결과 리스트 `totals = [0] * menu_count`를 준비한다. (여기서 `[0] * 4`는 내부 값이 정수이므로 9번의 `[[0]*3]*3`과 달리 참조 공유 문제가 생기지 않는다. 정수는 내부를 수정하는 연산 자체가 없는 불변 객체이기 때문이다.) 그 다음 각 행을 돌면서 그 행의 `j`번째 값을 `totals[j]`에 더해주면, 반복문이 끝났을 때 `totals`에는 각 열의 합계가 담긴다.

`len(sales[0])`으로 열 개수를 구했기 때문에 메뉴 개수가 바뀌어도 코드는 그대로 동작한다.

## 11번 (코드 작성)

아래는 학생 5명의 국어, 영어, 수학 점수를 2차원 리스트로 정리한 성적표이다. 각 행은 한 학생의 점수이고, 각 열은 왼쪽부터 순서대로 `[국어, 영어, 수학]`이다.

```python
scores = [
    [85, 70, 90],   # 0번 학생
    [72, 95, 88],   # 1번 학생
    [60, 80, 75],   # 2번 학생
    [98, 85, 92],   # 3번 학생
    [70, 77, 83],   # 4번 학생
]
```

성적표 전체에서 **가장 높은 점수를 받은 자리**를 찾아 `(학생번호, 과목번호)` 형태의 튜플로 반환하는 함수 `find_top(scores)`를 작성하시오. 최고점이 여러 군데에 있다면 더 앞쪽(작은 인덱스, 위→아래·왼→오 순서)에 있는 자리를 반환한다. 위 예시의 경우 `(3, 0)`이 반환되어야 한다 (3번 학생의 국어 점수 98이 전체 최고점).

**정답 예시**:

```python
def find_top(scores):
    best_i = 0
    best_j = 0
    best_value = scores[0][0]
    for i in range(len(scores)):
        for j in range(len(scores[i])):
            if scores[i][j] > best_value:
                best_value = scores[i][j]
                best_i = i
                best_j = j
    return (best_i, best_j)


scores = [
    [85, 70, 90],
    [72, 95, 88],
    [60, 80, 75],
    [98, 85, 92],
    [70, 77, 83],
]
print(find_top(scores))   # (3, 0)
```

**해설**:

1차원 리스트에서 최댓값의 위치를 찾을 때는 "지금까지의 최댓값"과 "그 위치" 두 변수만 관리하면 되지만, 2차원 리스트에서는 **위치가 행·열 두 축**으로 표현되므로 `best_i`, `best_j` 두 개의 좌표 변수가 필요하다.

초기 후보는 `scores[0][0]` 한 칸으로 설정한다. 이 값이 이미 후보이므로, 중첩 for문으로 모든 칸을 방문하면서 **더 큰 값을 만났을 때만** 최댓값과 좌표를 함께 갱신한다.

비교 조건으로 `>`(순수하게 크다)를 사용한 것이 중요하다. `>=`(크거나 같다)로 쓰면 같은 값이 뒤에서 또 나왔을 때 그쪽으로 덮어씌워져 "더 뒤에 있는 위치"가 반환되게 된다. 문제에서 요구한 "더 앞쪽에 있는 자리"를 보장하려면 이미 등장한 최댓값과 같은 값은 **갱신하지 않고 넘어가야** 하므로 `>`가 정답이다.

## 12번 (코드 작성)

어느 영화관의 3×4 좌석 배열이 아래와 같이 주어진다. `"X"`는 예약된 자리, `"O"`는 비어있는 자리이다.

```python
seats = [
    ["O", "X", "O", "O"],
    ["X", "O", "O", "X"],
    ["O", "O", "X", "O"],
]
```

**예약된 모든 자리의 좌표**를 `(행번호, 열번호)` 튜플 형태로 모아 리스트로 반환하는 함수 `find_reserved(seats)`를 작성하시오. 결과 리스트 안의 좌표는 위에서부터, 왼쪽부터의 순서로 정렬되어 있어야 한다. 위 예시의 경우 `[(0, 1), (1, 0), (1, 3), (2, 2)]`가 반환되어야 한다.

**정답 예시**:

```python
def find_reserved(seats):
    result = []
    for i in range(len(seats)):
        for j in range(len(seats[i])):
            if seats[i][j] == "X":
                result.append((i, j))
    return result


seats = [
    ["O", "X", "O", "O"],
    ["X", "O", "O", "X"],
    ["O", "O", "X", "O"],
]
print(find_reserved(seats))   # [(0, 1), (1, 0), (1, 3), (2, 2)]
```

**해설**:

1차원 순차 탐색(2번)의 자연스러운 2차원 확장이다. 다른 점은 두 가지이다.

- **순회 방식**: 중첩 for문으로 모든 칸을 방문해야 한다. 바깥 루프가 행 인덱스 `i`를 돌고, 안쪽 루프가 그 행의 열 인덱스 `j`를 도는 순서로 쓰면 **위→아래, 왼→오** 방향의 방문 순서가 자연스럽게 보장되므로 결과 리스트를 별도로 정렬할 필요가 없다.
- **반환 형식과 개수**: 단일 인덱스가 아니라 `(i, j)` 튜플로 위치를 표현해야 하고, 조건에 맞는 칸이 여러 개일 수 있다. 따라서 **한 번 찾았다고 즉시 `return`하지 않고** 빈 리스트 `result`에 좌표를 누적한 뒤, 반복문이 끝난 후 `result`를 반환한다.

`seats[i]`는 `i`번째 행(내부 리스트)이고, `len(seats[i])`는 그 행의 열 개수이다. 표의 가로 크기가 바뀌어도 코드가 그대로 동작한다.

## 13번 (코드 작성)

자연수 `a`와 0 이상의 정수 `n`을 입력으로 받아 `a`의 `n`제곱(즉 `a` × `a` × ... × `a`, `a`를 `n`번 곱한 값)을 반환하는 함수 `power(a, n)`을 **재귀 함수**로 작성하시오. 단, `n`이 0이면 결과는 1이다.

예를 들어 `power(2, 10)`은 `1024`, `power(3, 0)`은 `1`, `power(5, 3)`은 `125`를 반환해야 한다.

**조건**: 함수 내부에서 `for`나 `while` 같은 반복문을 사용하지 말고, 함수가 자기 자신을 호출하는 방식으로 작성할 것.

**정답 예시**:

```python
def power(a, n):
    if n == 0:
        return 1
    return a * power(a, n - 1)


print(power(2, 10))   # 1024
print(power(3, 0))    # 1
print(power(5, 3))    # 125
```

**해설**:

재귀 함수는 "자기 자신을 호출하는 함수"이며, 어떤 문제를 **한 단계 작은 같은 형태의 문제**로 줄여 푸는 발상으로 작성한다. 재귀 함수 설계의 두 축은 항상 같다.

1. **종료조건(base case)**: 더 이상 재귀를 부르지 않고 답을 곧장 반환하는 조건. 이게 없으면 함수가 영원히 자기 자신을 호출해 `RecursionError`가 발생한다.
2. **재귀 호출**: 입력을 한 단계 줄여 자기 자신을 호출하고, 그 결과를 사용해 현재 단계의 답을 만든다.

거듭제곱은 다음 관찰에서 자연스럽게 재귀가 나온다.

- `a`를 `n`번 곱한다는 것은 = `a`를 한 번 곱하고 + `a`를 `(n-1)`번 곱한다는 것과 같다.
- 식으로 쓰면 `power(a, n) = a * power(a, n-1)`.
- 이 식은 `n`을 1씩 줄여가다가 `n == 0`에서 멈춰야 한다. `n == 0`일 때의 답은 정의에 의해 1(임의의 수의 0제곱은 1).

코드의 흐름을 `power(2, 3)`으로 추적하면:

```
power(2, 3)
= 2 * power(2, 2)
= 2 * (2 * power(2, 1))
= 2 * (2 * (2 * power(2, 0)))
= 2 * (2 * (2 * 1))         # 종료조건에서 1 반환
= 2 * (2 * 2)
= 2 * 4
= 8
```

호출이 안쪽으로 깊이 들어갈 때마다 `n`이 1씩 줄고, `n == 0`에서 `1`이 튀어나오면서 그 값을 가지고 차례로 빠져나오며 곱이 누적된다. 종료조건을 `n == 1`에서 `return a`로 둬도 양수 `n`에 대해서는 동작하지만, 그러면 `n == 0`인 경우를 따로 처리해야 하므로 `n == 0` 종료가 더 깔끔하다.

## 14번 (코드 작성)

두 양의 정수 `a`, `b`를 입력으로 받아 두 수의 **최대공약수(GCD)** 를 반환하는 함수 `gcd(a, b)`를 **재귀 함수**로 작성하시오.

예를 들어 `gcd(48, 18)`은 `6`, `gcd(100, 75)`는 `25`, `gcd(17, 5)`는 `1`을 반환해야 한다.

**조건**: 함수 내부에서 `for`나 `while` 같은 반복문을 사용하지 말고, 자기 자신을 호출하는 방식으로 작성할 것.

**정답 예시**:

```python
def gcd(a, b):
    if b == 0:
        return a
    return gcd(b, a % b)


print(gcd(48, 18))    # 6
print(gcd(100, 75))   # 25
print(gcd(17, 5))     # 1
```

**해설**:

이 문제의 핵심은 유클리드 호제법(Euclidean algorithm)이라는 사실에 있다. 두 양의 정수 `a`, `b`의 최대공약수는 다음 성질을 만족한다.

- `a`를 `b`로 나눈 나머지를 `r`이라 할 때, `a`와 `b`의 GCD는 `b`와 `r`의 GCD와 같다. 즉 `gcd(a, b) = gcd(b, a % b)`.
- 이 성질을 한 번 적용할 때마다 두 번째 인자의 크기가 빠르게 줄어든다. 결국 어느 순간 두 번째 인자가 0이 되는데, `gcd(x, 0)`은 `x` 자신이다(0은 모든 정수의 배수이므로 `x`의 모든 약수가 둘 모두의 공약수).

이 두 식을 그대로 재귀 함수의 두 축으로 옮긴다.

1. **종료조건**: `b == 0`이면 `return a`.
2. **재귀 호출**: 그 외에는 `return gcd(b, a % b)`. 인자의 자리가 `(a, b) → (b, a % b)`로 한 칸씩 밀려간다는 점에 주의.

`gcd(48, 18)`의 호출 흐름을 추적하면:

```
gcd(48, 18)
= gcd(18, 48 % 18) = gcd(18, 12)
= gcd(12, 18 % 12) = gcd(12, 6)
= gcd(6,  12 % 6)  = gcd(6, 0)
= 6                # 종료조건: b == 0이므로 a 반환
```

처음 입력에서 `a < b`인 경우(예: `gcd(18, 48)`)도 한 번의 재귀 호출에서 `gcd(48, 18 % 48)` = `gcd(48, 18)`로 자연스럽게 자리가 바뀌므로 별도 처리가 필요 없다(`작은 수 % 큰 수 = 작은 수`이기 때문). 즉 입력 순서를 신경쓸 필요 없다.

비교를 통한 단순한 방법(1부터 `min(a, b)`까지 모두 시도하며 둘 다 나누는 가장 큰 수 찾기)도 답을 낼 수 있지만, 유클리드 호제법은 매 단계 두 번째 인자가 절반 이하로 줄어들기 때문에 비교 방식보다 압도적으로 빠르다(약 `log` 단계).

## 15번 (코드 변환)

아래는 1부터 `n`까지의 합(`1 + 2 + 3 + ... + n`)을 구하는 **재귀 함수**이다.

```python
def total_rec(n):
    if n == 0:
        return 0
    return n + total_rec(n - 1)


print(total_rec(10))   # 55
print(total_rec(100))  # 5050
```

이 함수와 **완전히 동일한 동작**을 하는 함수 `total(n)`을 **반복문(`for`)을 사용해서** 다시 작성하시오. 즉 어떤 `n`을 넣어도 두 함수의 반환값이 같아야 한다.

**정답 예시**:

```python
def total(n):
    s = 0
    for i in range(1, n + 1):
        s += i
    return s


print(total(10))    # 55
print(total(100))   # 5050
```

**해설**:

재귀 함수와 반복문은 같은 일을 다른 방식으로 표현한 것이다. 한쪽을 다른 쪽으로 변환할 때는 두 구조의 부품들이 어떻게 **대응**되는지 짚어내는 것이 핵심이다. 이 문제의 재귀 함수와 정답 for문 함수의 부품을 비교하면 다음과 같다.

| 재귀 함수의 부품 | for문 함수의 부품 | 의미 |
|---|---|---|
| 종료조건의 반환값 `return 0` | 누적 변수 초깃값 `s = 0` | "아무것도 더하지 않은 상태"의 시작값 |
| 재귀의 깊이 (`n`이 `n-1`, `n-2`, ..., 0으로 줄어듦) | 반복 횟수 (`i`가 1, 2, ..., `n`으로 늘어남) | 총 `n`번 호출 / 총 `n`번 반복 |
| 매 단계의 더해지는 값 `n` (재귀 호출 시점의 매개변수) | 매 반복의 더해지는 값 `i` (반복 변수) | 1부터 `n`까지의 모든 정수 |
| `return n + total_rec(n - 1)` | `s += i` | 현재 값을 누적값에 더함 |

재귀 호출 흐름을 펼쳐보면 `total_rec(3) = 3 + (2 + (1 + (0)))` = `3 + 2 + 1 + 0` 이고, for문 흐름은 `s = 0 → s = 0+1 → s = 1+2 → s = 3+3` = `0 + 1 + 2 + 3`이다. **더해지는 값들의 집합이 같다**(0, 1, 2, 3). 다만 재귀는 큰 값부터 더해 쌓고, for문은 작은 값부터 더해 쌓는다는 순서 차이만 있다. 덧셈은 교환·결합법칙이 성립하므로 결과는 같다.

변환 시 자주 하는 실수 두 가지에 주의한다.

- **누적 변수의 위치**: `s = 0`은 반드시 `for`문 **바깥**에 둬야 한다. 안에 두면 매 반복마다 0으로 초기화되어 누적이 안 된다. (재귀에서 종료조건이 호출 트리의 가장 깊은 곳에서 한 번만 실행되는 것과 대응된다.)
- **`range`의 끝값**: `range(1, n + 1)`로 써야 `n`까지 포함된다. `range(1, n)`은 `n-1`까지만 포함하므로 `n`이 빠진다. 재귀에서 마지막에 더해지는 값이 `1`(종료조건 직전)이고 가장 처음 더해지는 값이 `n`이라는 점과 정확히 매핑된다.

참고로 이 `total(n)`은 13차시에 학습한 함수와 동일한 형태이다.

## 16번 (객관식, 복수정답)

다음 코드를 실행한 상태에서 아래 ㄱ~ㄹ 중 옳은 것의 기호를 모두 쓰시오.

```python
t = ([1, 2], 3)
```

ㄱ. t는 불변(immutable) 객체이다.
ㄴ. 변수 t에 저장되어 있는 값들은 수정할 수 없다.
ㄷ. `del t[0]`을 실행하면 t는 `(3,)`이 된다.
ㄹ. `t[0] = [9, 9]`를 실행하면 t의 값은 `([9, 9], 3)`이 된다.

**정답**: ㄱ

**해설**:

이 문제의 핵심은 튜플의 불변성(immutability)이 정확히 무엇을 의미하는지를 묻는 것이다. "튜플은 변경 불가능하다"는 단순 암기로는 ㄴ을 옳다고 잘못 판단하기 쉽다.

- **ㄱ (옳음)**: 튜플은 불변 객체가 맞다. 즉, 일단 만들어진 튜플은 자신의 슬롯 개수(길이)나 각 슬롯이 가리키는 참조 대상을 바꿀 수 없다.

- **ㄴ (틀림, 함정)**: "t에 저장되어 있는 값"이 무엇인지 정확히 이해해야 한다. 튜플 `t = ([1, 2], 3)`에서 t의 슬롯에는 "리스트 객체에 대한 참조"와 "정수 3에 대한 참조"가 저장되어 있다. 이 **참조 자체**(슬롯)는 변경할 수 없지만, 슬롯이 **가리키는 객체가 mutable이라면 그 객체의 내부 상태는 얼마든지 변경할 수 있다**. 예를 들어 `t[0].append(9)`를 실행하면 에러 없이 동작하고 t는 `([1, 2, 9], 3)`이 된다. 즉 t에 저장된 값(리스트 객체)의 내부는 수정 가능하므로 ㄴ은 틀린 명제이다.

- **ㄷ (틀림)**: `del`로 튜플의 요소(슬롯)를 삭제하려는 시도이다. 튜플의 슬롯 개수는 immutable의 본질이므로 변경될 수 없고, `TypeError: 'tuple' object doesn't support item deletion`이 발생한다.

- **ㄹ (틀림)**: 튜플의 특정 슬롯에 새로운 객체를 재할당하려는 시도이다. 슬롯이 가리키는 참조 대상을 바꾸는 연산이므로 immutable의 원칙에 위배되고, `TypeError: 'tuple' object does not support item assignment`가 발생한다.

**정리**: 튜플의 불변성은 "튜플 자신의 구조(슬롯 개수와 각 슬롯이 가리키는 참조)가 변경되지 않는다"는 의미이지, "튜플을 통해 도달할 수 있는 모든 객체가 변경되지 않는다"는 의미가 아니다. 이 둘을 구분하지 못하면 ㄴ과 같은 명제에서 함정에 빠진다.

## 17번 (객관식, 복수정답)

다음 중 if 조건문에서 참(True)으로 판정되는 것을 모두 고르시오.

1. `0`
2. `""`
3. `"False"`
4. `[]`
5. `[0]`

**정답**: 3, 5

**해설**:

파이썬에서는 `if` 조건문에 들어가는 값이 자동으로 불(bool) 값으로 평가된다. 일반적인 자료형의 falsy(거짓으로 평가되는) 값은 다음과 같이 정해져 있다:

- 정수/실수: `0`, `0.0`
- 문자열: `""` (빈 문자열)
- 리스트/튜플/딕셔너리: `[]`, `()`, `{}` (빈 컨테이너)
- 그 외: `None`, `False`

이외의 값은 모두 truthy(참으로 평가)이다.

각 보기 분석:
- **1 (`0`)**: 정수 영. 위 규칙에 의해 falsy.
- **2 (`""`)**: 빈 문자열. falsy.
- **3 (`"False"`)**: 문자열 "False"는 길이 5의 **비어있지 않은** 문자열이다. 그 내용이 "False"라는 글자라는 사실은 truthy/falsy 판정과 무관하다. 따라서 truthy.
- **4 (`[]`)**: 빈 리스트. falsy.
- **5 (`[0]`)**: 요소 1개(`0`)를 가진 리스트. 리스트 자체는 비어있지 않으므로 truthy. **요소가 falsy 값이어도 컨테이너 자체의 비어있음 여부와는 무관**하다는 점이 핵심이다.

**함정 포인트**: 학생이 보기의 **내용**(글자가 "False"이거나 요소가 0)에 끌려가서 "거짓처럼 생겼으니 falsy"라고 잘못 판단하기 쉽다. 파이썬의 진리값 판정은 오직 **"그 자료형의 빈/영(empty/zero) 상태인가"** 만 본다.

## 18번 (객관식)

다음 코드의 실행 결과로 출력되는 값은?

```python
a = [[1, 2], [3, 4]]
b = a[:]
b[0][0] = 99
print(a)
```

1. `[[1, 2], [3, 4]]`
2. `[[99, 2], [3, 4]]`
3. `[[1, 2], [99, 4]]`
4. `[[99, 99], [99, 99]]`
5. TypeError 발생

**정답**: 2

**해설**:

`a[:]`(슬라이싱을 이용한 복사)는 **얕은 복사(shallow copy)** 를 수행한다. 얕은 복사는 바깥 리스트 객체는 새로 만들지만, 그 안에 들어있는 요소들은 **새로 복사하지 않고 참조만 그대로 가져다 넣는다**.

코드 실행 흐름을 메모리 관점에서 추적하면:

1. `a = [[1, 2], [3, 4]]`
   - 안쪽 리스트 두 개가 만들어진다. 편의상 `L1 = [1, 2]`, `L2 = [3, 4]`라 하자.
   - 바깥 리스트 `a`의 슬롯 0은 `L1`을, 슬롯 1은 `L2`를 가리킨다.

2. `b = a[:]`
   - **새로운 바깥 리스트** 객체가 만들어진다(즉 `a is b`는 False).
   - 그러나 새 바깥 리스트의 슬롯 0과 슬롯 1에는 `L1`, `L2`의 **참조가 그대로 복사**된다. 안쪽 리스트는 새로 만들어지지 않는다.
   - 결과적으로 `a[0]`과 `b[0]`은 동일한 `L1` 객체를, `a[1]`과 `b[1]`은 동일한 `L2` 객체를 가리킨다.

3. `b[0][0] = 99`
   - `b[0]`은 `L1`을 가리키므로, `L1`의 0번 슬롯이 99로 바뀐다. `L1`은 이제 `[99, 2]`.
   - `a[0]`도 같은 `L1`을 가리키고 있었으므로, `print(a)`를 하면 변경이 그대로 보인다.

따라서 출력은 `[[99, 2], [3, 4]]`.

**9번과의 관계**: 9번은 `[[0]*3]*3`처럼 **생성 시점의 `*` 반복**으로 참조가 공유되는 함정이었다. 18번은 **복사 시점**에 안쪽 객체가 함께 복사되지 않아 참조가 공유되는 함정이다. 트리거는 다르지만 "바깥 리스트의 슬롯들이 동일한 안쪽 리스트 객체를 가리킨다"는 본질은 같다.

**해결 방법**: 안쪽 객체까지 모두 복사하려면 **깊은 복사(deep copy)** 가 필요하다. `import copy` 후 `b = copy.deepcopy(a)`를 사용하거나, 컴프리헨션으로 `b = [row[:] for row in a]`를 쓰면 안쪽 리스트도 새로 복사되어 독립된 자료가 된다.

## 19번 (코드 작성)

정수 리스트 `data`와 양의 정수 `k`(단, `k <= len(data)`)가 주어진다. 리스트에서 **연속된 `k`개 원소의 합**이 최대가 되는 시작 인덱스와 그 합을 `(시작인덱스, 합)` 형태의 튜플로 반환하는 함수 `max_window_sum(data, k)`를 작성하시오. 최대 합인 구간이 여러 곳에 있다면 더 앞쪽(작은 인덱스)의 위치를 반환한다.

예시:

```python
data = [1, 5, 2, 8, 3, 4, 9, 1]
print(max_window_sum(data, 3))   # (4, 16)
```

각 윈도우의 합은 다음과 같다.

```
data[0:3] = [1, 5, 2]  합 8
data[1:4] = [5, 2, 8]  합 15
data[2:5] = [2, 8, 3]  합 13
data[3:6] = [8, 3, 4]  합 15
data[4:7] = [3, 4, 9]  합 16   <-- 최대
data[5:8] = [4, 9, 1]  합 14
```

따라서 시작 인덱스 `4`, 합 `16`을 묶은 `(4, 16)`이 정답이다.

**정답 예시**:

```python
def max_window_sum(data, k):
    best_idx = 0
    best_sum = sum(data[:k])
    for i in range(1, len(data) - k + 1):
        s = sum(data[i:i+k])
        if s > best_sum:
            best_sum = s
            best_idx = i
    return (best_idx, best_sum)


print(max_window_sum([1, 5, 2, 8, 3, 4, 9, 1], 3))   # (4, 16)
```

**해설**:

이 문제는 **슬라이딩 윈도우(sliding window)** 라 불리는 패턴이다. 길이가 `k`인 창문(window)을 리스트의 왼쪽 끝부터 한 칸씩 오른쪽으로 밀면서, 매 위치에서 창문 안의 합을 계산해 최댓값을 갱신해 나간다.

설계의 두 가지 핵심:

1. **윈도우의 시작 인덱스 범위**: 시작 인덱스 `i`에서 `data[i:i+k]`를 잘라 쓴다. 마지막 윈도우의 끝 인덱스는 `len(data)`를 넘으면 안 되므로 `i + k <= len(data)`, 즉 `i`는 최대 `len(data) - k`까지이다. for문에서 `range(1, len(data) - k + 1)`로 쓴 것이 이 상한을 반영한 것. **`+1`을 잊으면 마지막 윈도우 하나가 누락**된다는 점이 흔한 실수.

2. **추적 변수 2개의 동시 갱신**: 최대 합 자체(`best_sum`)와 그 위치(`best_idx`)를 함께 관리해야 한다. 11번 `find_top`에서 다룬 패턴과 동일한 구조이다. 초기 후보로 0번 위치의 윈도우(`data[:k]`)를 잡아두고, 이후 더 큰 합이 나타날 때만 두 변수를 함께 갱신한다.

**`>` vs `>=`**: 비교에 `>`를 써야 "동률이면 더 앞쪽" 조건이 보장된다. `>=`로 쓰면 같은 합이 뒤에서 또 나왔을 때 그쪽으로 덮어쓰여 더 뒤의 위치가 반환된다. 11번 해설의 원리가 그대로 적용된다.

**효율성 참고**: 위 정답은 매 위치마다 `sum(data[i:i+k])`를 새로 계산하므로 전체 시간이 약 `n*k` 정도이다. 더 빠르게 하려면 직전 윈도우 합에서 **빠지는 원소(`data[i-1]`)를 빼고 새로 들어오는 원소(`data[i+k-1]`)를 더하는 방식**으로 한 번의 덧셈/뺄셈만으로 다음 합을 구할 수 있다(시간 약 `n`). 이를 진짜 의미의 슬라이딩 윈도우 최적화라고 부른다.

```python
def max_window_sum(data, k):
    cur = sum(data[:k])
    best_idx, best_sum = 0, cur
    for i in range(1, len(data) - k + 1):
        cur = cur - data[i-1] + data[i+k-1]
        if cur > best_sum:
            best_sum, best_idx = cur, i
    return (best_idx, best_sum)
```

리스트 길이가 작으면 두 방식의 차이는 체감이 안 되지만, 데이터가 많아질수록 차이가 커진다.

## 20번 (코드 작성)

분식집의 메뉴와 가격을 담은 딕셔너리 `menu`와 사용자의 초기 예산 `budget`이 주어진다. 사용자가 메뉴명을 한 줄씩 입력하는 방식으로 주문을 받는 함수 `order(menu, budget)`을 작성하시오. 함수는 다음 규칙을 따라야 한다.

1. 사용자가 `'끝'`을 입력하면 주문을 종료한다.
2. 입력한 이름이 `menu`에 없는 경우 `"없는 메뉴"`를 출력하고 다음 입력으로 넘어간다(주문 처리 X).
3. 입력한 메뉴의 가격이 현재 잔액보다 크면 `"예산 부족"`을 출력하고 다음 입력으로 넘어간다(주문 처리 X).
4. 위 둘에 해당하지 않으면 정상 주문 처리: 잔액에서 가격을 차감하고, 주문 내역에 메뉴명을 추가하며, `"<메뉴명> 주문 완료 (잔액 <금액>원)"` 형식으로 출력한다.

함수는 종료 시 `(주문내역_리스트, 잔액)` 튜플을 반환한다.

예시 호출:

```python
menu = {'떡볶이': 5000, '라면': 3000, '순대': 4000, '콜라': 1500}
result = order(menu, 10000)
print(result)
```

예시 입력 → 화면 출력:

```
입력: 떡볶이      → 떡볶이 주문 완료 (잔액 5000원)
입력: 라면        → 라면 주문 완료 (잔액 2000원)
입력: 김밥        → 없는 메뉴
입력: 순대        → 예산 부족
입력: 콜라        → 콜라 주문 완료 (잔액 500원)
입력: 끝          → (종료)
```

위 시나리오에서 함수 반환값은 `(['떡볶이', '라면', '콜라'], 500)` 이다.

**정답 예시**:

```python
def order(menu, budget):
    bought = []
    while True:
        m = input()
        if m == '끝':
            break
        if m not in menu:
            print("없는 메뉴")
            continue
        if menu[m] > budget:
            print("예산 부족")
            continue
        budget -= menu[m]
        bought.append(m)
        print(f"{m} 주문 완료 (잔액 {budget}원)")
    return (bought, budget)


menu = {'떡볶이': 5000, '라면': 3000, '순대': 4000, '콜라': 1500}
print(order(menu, 10000))
```

**해설**:

이 문제는 **`while True` + `break` + `continue` + 다중 분기**를 한 번에 활용하는 종합 제어문 문제이다. 학생이 한 가지 분기만 빠뜨려도 무한루프, KeyError, 잔액 음수 등으로 명확히 드러난다.

핵심 설계 포인트는 다음과 같다.

1. **`while True`로 무한 입력 받기**: 입력 횟수가 미리 정해져 있지 않으므로 `for`보다 `while`이 자연스럽다. 종료 시점은 사용자가 `'끝'`을 입력했을 때 한 군데뿐이며, 그 자리에서 `break`로 빠져나온다.

2. **검사 순서가 중요하다**: 분기 조건을 다음 **순서대로** 써야 한다.
   - (a) 종료 신호(`'끝'`) 검사 — 가장 먼저
   - (b) 메뉴 존재 검사(`m not in menu`)
   - (c) 예산 검사(`menu[m] > budget`)
   - (d) 정상 주문 처리
   
   특히 (b)를 (c)보다 먼저 해야 한다. 순서를 바꿔서 `menu[m]`을 먼저 조회하면, 없는 메뉴 입력 시 KeyError가 발생한다. (이는 7차시 딕셔너리 단원에서 다룬 "없는 키 접근은 KeyError" 함정과 연결된다.)

3. **`continue`의 역할**: (b)와 (c)에서 조건이 맞으면 그 회차의 나머지 코드(주문 처리)를 건너뛰고 다음 입력을 받아야 한다. 이때 쓰는 키워드가 `continue`이다. **`break`로 잘못 쓰면** 첫 번째 잘못된 입력에서 함수가 즉시 끝나버린다.

4. **누적 변수의 위치**: `bought = []`는 `while` **바깥**에 둬야 누적이 유지된다. 안에 두면 매 회차마다 빈 리스트로 초기화된다(15번 해설의 누적 변수 위치 원칙과 동일). `budget`은 매개변수로 받은 후 함수 내부에서만 차감하므로 호출자의 변수에는 영향이 없다(파이썬은 정수가 immutable이라 함수 안의 `budget -= ...`은 지역 변수 재할당으로 처리됨).

5. **`continue` 사용 시 주의**: 9차시 단원의 핵심 함정인 "continue 직전에 증감/입력 갱신 빠뜨려서 무한루프"는 이 문제에서는 입력이 매 회차 `m = input()`으로 새로 들어오기 때문에 자연스럽게 회피된다. 다만 `while`의 입력문이 어디 있는지 학생이 헷갈리면 빠질 수 있는 함정이다.

**확장 과제(생각해 볼 거리)**: 같은 메뉴를 여러 번 주문할 수 있도록 허용되어 있다. 만약 "각 메뉴는 1회만 주문 가능"이 추가 조건이라면 `if m in bought:` 검사를 한 단계 더 끼워 넣으면 된다. 분기 조건의 순서와 위치를 재설계하는 연습이 된다.

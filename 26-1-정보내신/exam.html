정수 리스트 data와 양의 정수 k(단, k <= len(data))가 주어진다. 리스트에서 연속된 k개 원소의 합이 최대가 되는 시작 인덱스와 그 합을 (시작인덱스, 합) 형태의 튜플로 반환하는 함수 max_window_sum(data, k)를 작성하시오. 최대 합인 구간이 여러 곳에 있다면 더 앞쪽(작은 인덱스)의 위치를 반환한다.

예시:

data = [1, 5, 2, 8, 3, 4, 9, 1] print(max_window_sum(data, 3)) # (4, 16)

각 윈도우의 합은 다음과 같다.

참고data[0:3] = [1, 5, 2] 합 8 data[1:4] = [5, 2, 8] 합 15 data[2:5] = [2, 8, 3] 합 13 data[3:6] = [8, 3, 4] 합 15 data[4:7] = [3, 4, 9] 합 16 <-- 최대 data[5:8] = [4, 9, 1] 합 14

따라서 시작 인덱스 4, 합 16을 묶은 (4, 16)이 정답이다.

분식집의 메뉴와 가격을 담은 딕셔너리 menu와 사용자의 초기 예산 budget이 주어진다. 사용자가 메뉴명을 한 줄씩 입력하는 방식으로 주문을 받는 함수 order(menu, budget)을 작성하시오. 함수는 다음 규칙을 따라야 한다.

  1. 사용자가 '끝'을 입력하면 주문을 종료한다.
  2. 입력한 이름이 menu에 없는 경우 "없는 메뉴"를 출력하고 다음 입력으로 넘어간다(주문 처리 X).
  3. 입력한 메뉴의 가격이 현재 잔액보다 크면 "예산 부족"을 출력하고 다음 입력으로 넘어간다(주문 처리 X).
  4. 위 둘에 해당하지 않으면 정상 주문 처리: 잔액에서 가격을 차감하고, 주문 내역에 메뉴명을 추가하며, "<메뉴명> 주문 완료 (잔액 <금액>원)" 형식으로 출력한다.

함수는 종료 시 (주문내역_리스트, 잔액) 튜플을 반환한다.

예시 호출:

menu = { '떡볶이': 5000, '라면': 3000, '순대': 4000, '콜라': 1500, } result = order(menu, 10000) print(result)

예시 입력 / 화면 출력:

시나리오입력: 떡볶이 -> 떡볶이 주문 완료 (잔액 5000원) 입력: 라면 -> 라면 주문 완료 (잔액 2000원) 입력: 김밥 -> 없는 메뉴 입력: 순대 -> 예산 부족 입력: 콜라 -> 콜라 주문 완료 (잔액 500원) 입력: 끝 -> (종료)

위 시나리오에서 함수 반환값은 (['떡볶이', '라면', '콜라'], 500) 이다.

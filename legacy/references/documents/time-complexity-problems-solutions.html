시간복잡도 분석 문제 - 해설

정답 및 상세 설명
문제 1. 배열 최댓값 찾기
def find_max(arr): max_val = arr[0] for i in range(1, len(arr)): if arr[i] > max_val: max_val = arr[i] return max_val
정답: O(n)
해설:
• 반복문 분석: for 루프가 배열 전체를 한 번 순회 (n-1번)
• 내부 연산: 비교 연산 1회 (O(1))
• 총 연산: (n-1) × O(1) = O(n-1) ≈ O(n)
• 핵심: 배열의 모든 원소를 확인해야 하므로 입력 크기에 비례
문제 2. 상수 내부 반복 (함정 문제)
def process_data(n): for i in range(n): for j in range(100): print(i, j)
정답: O(n)
해설:
• 외부 반복: n번
• 내부 반복: 100번 (상수!)
• 총 연산: n × 100 = 100n
• Big-O 규칙 적용: 상수 계수 무시 → O(n)
• 주의: 내부 반복이 상수이므로 O(n²)이 아님!
문제 3. 이중 중첩 반복문
def print_pairs(arr): for i in range(len(arr)): for j in range(len(arr)): print(arr[i], arr[j])
정답: O(n²)
해설:
• 외부 반복: n번
• 내부 반복: 각각 n번
• 총 연산: n × n = n²
• 결과: O(n²)
• 특징: 전형적인 이차 시간 복잡도, 모든 원소 쌍을 출력
문제 4. 삼각형 패턴 (줄어드는 중첩)
def triangle_sum(n): total = 0 for i in range(n): for j in range(i + 1): total += 1 return total
정답: O(n²)
해설:
• 반복 횟수: 1 + 2 + 3 + ... + n번
• 등차수열의 합: n(n+1)/2 = (n² + n)/2
• 최고차항: n²/2
• Big-O 규칙: 상수(1/2) 무시 → O(n²)
• 핵심: 내부 반복이 점차 증가하지만 여전히 O(n²)
문제 5. 이진 탐색
def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1
정답: O(log n)
해설:
• 핵심 동작: 매 반복마다 탐색 범위가 절반으로 줄어듦
• 반복 횟수: n → n/2 → n/4 → ... → 1
• 단계 수: log₂(n)번
• 결과: O(log n)
• 조건: 배열이 정렬되어 있어야 함
• 예시: n=1024일 때 최대 10번만 비교
문제 6. 소수 판별 (최적화)
def is_prime(n): if n < 2: return False for i in range(2, int(n**0.5) + 1): if n % i == 0: return False return True
정답: O(√n)
해설:
• 반복 범위: 2부터 √n까지
• 반복 횟수: √n - 1 ≈ √n
• 이유: n = a × b에서 a ≤ √n ≤ b 이므로 √n까지만 확인하면 됨
• 결과: O(√n)
• 비교: O(n)보다 훨씬 효율적 (n=10000일 때 100번만 검사)
문제 7. 소수 개수 세기 (비효율적)
def count_primes(n): count = 0 for i in range(2, n + 1): is_prime = True for j in range(2, i): if i % j == 0: is_prime = False break if is_prime: count += 1 return count
정답: O(n²)
해설:
• 외부 반복: n번 (2부터 n까지)
• 내부 반복: 평균 i/2번 (최악 i-1번)
• 총 연산: 2 + 3 + 4 + ... + n ≈ n²/2
• 결과: O(n²)
• 개선 가능: 내부를 √i까지만 체크하면 O(n√n)으로 개선
문제 8. 피보나치 재귀 (순수 재귀)
def fibonacci(n): if n <= 1: return n return fibonacci(n - 1) + fibonacci(n - 2)
정답: O(2ⁿ)
해설:
• 재귀 패턴: 매번 2개로 분기 (fibonacci(n-1) + fibonacci(n-2))
• 재귀 트리: 높이 n, 각 레벨에서 2배씩 증가
• 호출 횟수: 1 + 2 + 4 + 8 + ... + 2ⁿ ≈ 2ⁿ⁺¹
• 결과: O(2ⁿ)
• 특징: 매우 비효율적! 같은 값을 여러 번 계산
• 예시: n=30일 때 약 10억 번 호출
• 개선: 메모이제이션 사용 시 O(n)으로 개선 가능
문제 9. 이진 트리 순회 (재귀)
def tree_traversal(n): if n <= 0: return tree_traversal(n // 2) tree_traversal(n // 2)
정답: O(n)
해설:
• 재귀 패턴: 2번 호출하지만 n이 절반씩 줄어듦
• 트리 깊이: log₂(n) (절반씩 나누므로)
• 각 레벨 노드 수:
  - 레벨 0: 1개 (n)
  - 레벨 1: 2개 (n/2 × 2)
  - 레벨 2: 4개 (n/4 × 4)
  - ...
  - 레벨 log n: n개
• 총 노드 수: 1 + 2 + 4 + ... + n = 2n - 1
• 결과: O(n)
• 핵심: 2^(log₂ n) = n
문제 10. 모든 부분집합 생성
def all_subsets(arr, index=0): if index == len(arr): return [[]] without = all_subsets(arr, index + 1) with_elem = [] for subset in without: with_elem.append([arr[index]] + subset) return without + with_elem
정답: O(2ⁿ)
해설:
• 핵심 개념: 각 원소마다 "포함" 또는 "제외" 2가지 선택
• 부분집합 개수: 2ⁿ개
• 재귀 구조: 각 원소에 대해 without과 with_elem 생성
• 결과: O(2ⁿ)
• 예시:
  - n=3: {}, {a}, {b}, {c}, {a,b}, {a,c}, {b,c}, {a,b,c} = 8개
  - n=10: 1024개
• 주의: n이 조금만 커져도 매우 느려짐

복잡도 요약

O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
핵심 패턴:
• 상수 또는 직접 접근 → O(1)
• 절반씩 나누기/증가 → O(log n)
• 제곱근까지 반복 → O(√n)
• 단일 반복문 → O(n)
• 분할정복 (병합정렬 등) → O(n log n)
• 이중 중첩 반복 → O(n²)
• 2갈래 재귀 (피보나치, 부분집합) → O(2ⁿ)

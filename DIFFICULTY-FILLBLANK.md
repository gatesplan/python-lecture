# 빈칸채우기 문제 난이도 기준

난이도 스코어 = 빈칸수 x 종속성 x 위치역할 x 코드복잡도

## 축 1: 빈칸 수

| 빈칸 | 계수 |
|------|------|
| 1칸 | x1.0 |
| 2칸 | x1.5 |
| 3칸 | x2.0 |

## 축 2: 빈칸 간 종속성

| 종속성 | 계수 | 설명 |
|--------|------|------|
| 독립 | x1.0 | 각 빈칸이 별개 줄에서 별개 역할 |
| 순차 종속 | x1.5 | 빈칸1의 변수를 빈칸2에서 사용 |
| 상호 종속 | x2.0 | 두 빈칸이 함께 맞아야 로직 성립 |

## 축 3: 빈칸 위치/역할

| 위치 | 계수 |
|------|------|
| 함수 인자/파라미터 | x0.8 |
| 단순 식 | x1.0 |
| 반복 조건부 (range 인자, while 조건) | x1.2 |
| 조건식 전체 | x1.2 |
| 대입 우변 | x1.2 |
| break/continue 조건 블럭 (멀티라인) | x1.5 |
| 반복 내부 로직 (멀티라인) | x1.8 |
| 복합 식 | x1.8 |
| 조건 분기 전체 (멀티라인) | x1.8 |

### 빈칸 크기 분류

- **인라인**: `[[]]` - 식, 조건, 인자 등 한 줄 내 부분
- **멀티라인**: `[[[N]]]` - 블럭 단위 (반복 body, 조건 분기 등) *컴포넌트 미구현*

## 축 4: 코드 복잡도

| 구조 | 계수 |
|------|------|
| 단순 순차 | x1.0 |
| 단일 반복 | x1.2 |
| 조건+반복 | x1.5 |
| 중첩 반복 | x1.8 |

## 제외 항목

- **내용 제약** (함수, 클래스 등): 난이도 축이 아닌 출제 범위 필터로 별도 관리
- **단서 수준**: 변수명은 비서술적이 기본, 힌트 없음이 기본이므로 축에서 제외

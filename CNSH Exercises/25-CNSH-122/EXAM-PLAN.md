## 📋 파일명 규칙

```
problems-cnsh122-{n}-{title}.xml
```

- `{n}`: 두 자리 숫자 (01, 02, 03, ...)
- `{title}`: 영문 소문자, 하이픈으로 구분

예시: `problems-cnsh122-01-basic-recursion.xml`

---

## 🎨 문제 유형 (총 11가지)

### 기존 유형 (4가지)
1. **빈칸 채우기** - `____` 형태
2. **블록 채우기** - `# 여기를 채우세요`
3. **전체 작성** - `pass`로 시작
4. **서술형** - 개념 설명

### 신규 유형 (7가지) ⭐ NEW
5. **디버깅** - 잘못된 코드 찾아 고치기
6. **트레이싱** - 실행 과정 추적 (호출 횟수 등)
7. **출력 예측** - 코드 보고 결과 예측
8. **코드 비교** - 효율성/정확성 비교
9. **조건 역추론** - 원하는 출력 만드는 입력 찾기
10. **순서 맞추기** - 뒤섞인 코드 줄 정렬
11. **복잡도 분석** - Big-O 표기법으로 분석

### 유형별 적용 원칙
- **출력 예측**: 모든 파일 워밍업용 (1개)
- **디버깅**: 거의 모든 파일 (1-2개)
- **복잡도 분석**: 주로 Level 6, 일부 다른 파일
- **트레이싱**: 재귀 파일 중심
- **코드 비교**: 최적화가 있는 주제
- **조건 역추론**: 수학적 문제
- **순서 맞추기**: 구조가 명확한 문제

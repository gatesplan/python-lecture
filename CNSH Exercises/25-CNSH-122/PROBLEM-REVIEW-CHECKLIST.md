# Problem XML 파일 검토 체크리스트

**작성일**: 2025-11-09
**목적**: 제작한 문제 XML 파일의 품질을 체계적으로 검토

---

## ✅ 1. 형식 규칙 준수 (Form)

### XML 구조
- [ ] XML 선언 포함 (`<?xml version="1.0" encoding="UTF-8"?>`)
- [ ] `<problems>` 루트 태그로 감싸짐
- [ ] 각 문제가 `<problem>` 태그로 감싸짐
- [ ] 모든 태그 정상 닫힘

### CDATA 사용
- [ ] `<code>` 내용이 `<![CDATA[...]]>`로 감싸짐
- [ ] `<input>` 내용이 `<![CDATA[...]]>`로 감싸짐 (있는 경우)
- [ ] `<output>` 내용이 `<![CDATA[...]]>`로 감싸짐 (있는 경우)
- [ ] `<solution>` 내용이 `<![CDATA[...]]>`로 감싸짐

### 코드 스타일
- [ ] 함수명이 `f`로 단순화됨
- [ ] 변수명이 1-2글자로 단순화됨 (`n`, `k`, `x`, `y`, `i`, `j` 등)
- [ ] 빈칸이 `____` (언더스코어 정확히 4개)로 표기
- [ ] 들여쓰기 4칸 스페이스 일관적 사용

### 주석 규칙
- [ ] **문제 코드**에 주석이 **없음** (철저히 확인!)
- [ ] **solution**에만 주석 포함
- [ ] hint 필드가 비어있음 (사용 안 함)

---

## ✅ 2. 내용 정확성 (Correctness)

### 문제 설명 (description)
- [ ] 무엇을 해야 하는지 명확함
- [ ] 1-3문장으로 간결함
- [ ] 모호한 표현 없음 ("적절히", "알맞게" 등 지양)
- [ ] 한글 문법 오류 없음

### 코드 정확성
- [ ] Python 문법 오류 없음
- [ ] 실행 가능한 코드
- [ ] 들여쓰기 오류 없음
- [ ] 변수명 오타 없음

### Input/Output 정합성
- [ ] `<input>`이 실행 가능한 Python 코드
- [ ] `<output>`이 실제 실행 결과와 **정확히 일치**
- [ ] 줄바꿈 등 공백 문자도 일치
- [ ] 여러 테스트 케이스의 출력 순서 일치

### Solution 정확성
- [ ] 정답 코드가 문법적으로 올바름
- [ ] 정답 코드가 실제로 작동함
- [ ] 빈칸 문제의 정답이 명시됨 (`# 정답: ...`)
- [ ] 해설 내용이 정확함
- [ ] 해설이 충분히 상세함 (왜 그런지 설명)

---

## ✅ 3. 구성 및 난이도 (Structure)

### 문제 개수
- [ ] 계획서의 문제 개수와 일치
- [ ] 권장 범위 내 (6-10개)

### 유형 분포
- [ ] 계획서의 유형 분포와 일치
- [ ] 출력 예측이 첫 문제로 배치 (워밍업)
- [ ] 디버깅 문제 포함 (1-2개)
- [ ] 다양한 유형 골고루 분포

### 난이도 순서
- [ ] 쉬운 문제부터 시작 (출력 예측 또는 빈칸 1개)
- [ ] 점진적으로 난이도 증가
- [ ] 어려운 문제가 후반부 배치 (전체 작성, 복잡한 디버깅)
- [ ] 급격한 난이도 점프 없음

### 주제 일관성
- [ ] 파일명의 주제와 일치
- [ ] 모든 문제가 같은 개념 범위 내
- [ ] 문제 간 자연스러운 연결

---

## ✅ 4. 학습 효과 (Pedagogy)

### 점진적 학습
- [ ] 같은 코드의 다른 부분을 채우는 문제로 반복 학습
- [ ] 개념 확인 → 적용 → 심화 순서
- [ ] 이전 문제가 다음 문제의 힌트 역할

### 실수 포인트
- [ ] 디버깅 문제가 학생들이 자주 하는 실수 포함
- [ ] 논리 오류 > 문법 오류 (학습 효과 높음)
- [ ] 너무 명백한 오류는 지양 (학습 효과 낮음)

### 개념 이해
- [ ] 서술형 문제로 개념 확인
- [ ] 트레이싱 문제로 실행 과정 이해
- [ ] 코드 비교 문제로 최적화 감각 향상

---

## ✅ 5. 실용성 (Practicality)

### 시험 적합성
- [ ] 손으로 추적 가능한 범위의 입력
- [ ] 10-15분 내 해결 가능한 난이도
- [ ] 채점 기준이 명확함
- [ ] 부분 점수 가능한 구조

### 코드 복잡도
- [ ] 출력 예측: 5-10줄 이내
- [ ] 트레이싱: 재귀 깊이 5 이내
- [ ] 빈칸: 1-3개 정도
- [ ] 전체 작성: 10-15줄 이내

### 실행 가능성
- [ ] 모든 코드를 실제로 실행해봄
- [ ] input/output이 정확히 일치 확인
- [ ] 엣지 케이스 고려 (0, 1, 음수 등)

---

## ✅ 6. 최종 검증 (Final Check)

### Problem Renderer 테스트
- [ ] Problem Renderer에서 정상 렌더링
- [ ] 레이아웃 깨짐 없음
- [ ] 코드 박스 정상 표시
- [ ] 입출력 박스 정상 표시

### 직접 풀이
- [ ] 각 문제를 종이에 직접 풀어봄
- [ ] 예상 풀이 시간 측정
- [ ] 애매한 부분 없는지 확인
- [ ] 설명 추가 필요 여부 판단

### 중복 확인
- [ ] 같은 파일 내 중복 문제 없음
- [ ] 다른 파일과 중복 없음
- [ ] 유사 문제는 의도적 반복 학습용

### 오타 및 문법
- [ ] 모든 description 오타 확인
- [ ] 모든 solution 오타 확인
- [ ] 주석 내용 문법 확인
- [ ] 변수명 일관성 확인

## 🔧 사용 방법

### 파일 제작 후
1. 이 체크리스트를 출력하거나 복사
2. 위에서부터 순서대로 체크
3. 하나라도 체크 안 되면 수정
4. 모두 체크되면 완료

### 우선순위
- **필수 체크 6개**: 하나라도 실패하면 수정 필수
- **중요 체크 4개**: 대부분 만족해야 함
- **권장 체크 4개**: 시간 여유 시 확인

# 26-2 정보수업 - 네트워크와 사물인터넷 (260530)

이번 주 이론 수업은 네트워크와 사물인터넷(IoT). 손글씨 필기(note1)를 디지털 노트로 정리한 내용이다.

## 1. 네트워크

### IP 주소
컴퓨터(기기)를 구분할 수 있는 고유한 숫자. 인터넷에서 각 기기의 주소 역할을 한다.

- **IPv4**: 8비트씩 4자리 = 총 32비트 -> 약 2^32개(약 43억 개)의 주소
- 기기 수가 늘어 주소가 부족해짐 -> **IPv6(128비트)** 로 전환 중. 2^128개로 사실상 무한

### IP로 기기를 찾아가는 과정
IP 주소는 단순한 번호가 아니라 **위치를 담은 주소**다. 그래서 택배가 거점을 거쳐 배달되듯 단계적으로 목적지를 찾아간다.

- **IP는 두 부분으로 나뉨**: 네트워크 부분(어느 동네인가) + 호스트 부분(그 동네 몇 호인가)
  - 비유: "서울시 강남구"(네트워크 부분) + "테헤란로 123, 4층"(호스트 부분)
- **라우터가 거점을 거쳐 전달 (라우팅)**: 인터넷은 라우터로 연결된 수많은 네트워크. 각 라우터가 목적지 IP의 네트워크 부분을 보고 다음 경로로 넘겨, 목적지 네트워크까지 점점 가까워진다
  - 비유: [동네 영업소] -> [지역 허브] -> [목적지 영업소]
- **도착 후 실제 기기에 전달**: 목적지 네트워크까지는 IP로 찾아오고, 마지막엔 기기마다 고유한 물리 주소(**MAC 주소**)로 전달한다

한 줄 정리: **IP(위치)로 동네까지, MAC(기기 본인)으로 문 앞까지.**

### 도메인 이름과 DNS
- **도메인 이름**: 외우기 어려운 숫자 IP를 사람이 기억하기 쉬운 이름으로 바꾼 것
  - 비유: 연락처에 전화번호 대신 "엄마", "친구" 같은 이름을 저장하는 것과 같음
- **DNS 서버**: 도메인 이름을 입력하면 그에 맞는 IP 주소로 바꿔주는 서버
  - 예) `www.naver.com` 을 입력하면 DNS가 대응하는 IP 주소를 찾아 연결해 줌 (도메인 <-> IP)

### 도메인 구조
예) `www.yahoo.co.kr`

| 부분 | 의미 |
|---|---|
| www | 호스트 이름 (웹 서비스를 가리키는 관례적 이름) |
| yahoo | 기관(서비스) 이름 |
| co | 성격: 기관/회사 (공공기관은 go) |
| kr | 국가: 대한민국 |

(주의: 필기에는 `www`를 "프로토콜"로 적었으나 정확하지 않다. 자세한 정정은 `1-내용보강.md` 참고)

### 네트워크의 종류: 유선 vs 무선
- **유선**: 속도가 빠르고 신뢰할 수 있음. 단, 이동이 어렵고 선이 복잡함
- **무선**: 물리적 연결 없이 이동이 편리함. 단, 보안에 취약하고 거리 제한 및 신호 간섭이 있음

### 로컬 환경 공유 시 보안 원칙
공유 폴더, 에어드랍 등으로 가까운 기기끼리 파일을 주고받을 때 지켜야 할 4가지.

1. **최소 공유**: 꼭 필요한 것만 공유한다
2. **사전 검사**: 공유하기 전에 파일과 공유 대상을 점검한다
3. **정보 보호**: 민감한 정보가 새지 않도록 보호한다
4. **권한 설정**: 접근 권한을 적절히 제한한다

## 2. 사물인터넷 (IoT)

**IoT = Internet of Things**. 사물에 센서와 통신 기능을 넣어 인터넷에 연결하고, 데이터를 주고받게 하는 기술.

### 구성 요소 D / N / P

| 요소 | 이름 | 역할 | 예시 |
|---|---|---|---|
| **D** | Device (센서) | 데이터를 수집하는 기기 | 마이크로컨트롤러, 각종 센서 |
| **N** | Network (네트워크) | 수집한 데이터를 전달하는 통신 기술 | NFC, RFID, Beacon, ZigBee, WiFi |
| **P** | Platform (플랫폼) | 데이터를 모아 유용한 정보로 가공/생산 | 데이터 수집/분석 서버 |

### IoT 네트워크 기술 (N 요소)
N(네트워크)에 쓰이는 대표 통신 기술들. 각자 거리와 용도가 다르다.

- **NFC** (Near Field Communication, 근거리 무선통신): 약 10cm 이내 초근거리에서 갖다 대면 바로 통신. 예) 교통카드, 스마트폰 모바일 결제, 출입 태그
- **RFID** (Radio Frequency Identification, 무선 인식): 전파로 태그에 담긴 정보를 읽는 기술. 수동형 태그는 자체 배터리 없이 리더기 전파로 동작. 예) 물류/재고 관리, 출입증, 하이패스. (NFC는 RFID를 근거리 양방향용으로 발전시킨 것)
- **Beacon** (비콘): 저전력 블루투스로 짧은 신호를 주기적으로 "방송"해 근처 기기에 위치/알림을 전달. 예) 매장/전시관 실내 위치 안내, 가까이 가면 쿠폰 알림
- **ZigBee** (지그비): 저전력으로 가까운 기기끼리 그물망처럼 연결하는 통신. 속도는 느리지만 전력 소모가 적어 오래간다. 예) 스마트홈 센서(조명, 온도, 도어센서) 네트워크
- **WiFi**: 인터넷 공유기에 연결해 빠르고 넓게 통신. 예) 노트북/폰 인터넷, 대용량 데이터 전송

(다섯 기술의 거리/속도/전력 비교표와 "WiFi가 어떤 점에서 뛰어난가"에 대한 답은 `1-내용보강.md` 참고)

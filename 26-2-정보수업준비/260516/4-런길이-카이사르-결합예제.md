# 런길이 + 카이사르 결합 예제

## 사용 코드

```python
def rle(s):
    result = ""
    count = 1
    for i in range(len(s)):
        if i == len(s) - 1 or s[i] != s[i+1]:
            result += s[i] + str(count)
            count = 1
        else:
            count += 1
    return result


def caesar(text, shift):
    result = ""
    for ch in text:
        if 'A' <= ch <= 'Z':
            result += chr((ord(ch) - ord('A') + shift) % 26 + ord('A'))
        elif 'a' <= ch <= 'z':
            result += chr((ord(ch) - ord('a') + shift) % 26 + ord('a'))
        else:
            result += ch
    return result
```

## 1부. 런 길이 부호화 단독 예측

### 문제 1-1
다음 코드의 출력은?
```python
print(rle("AABBBCCCC"))
```

**정답**: `A2B3C4`

**풀이**: A 2번, B 3번, C 4번 연속.

### 문제 1-2
다음 코드의 출력은?
```python
print(rle("ZZZAAB"))
```

**정답**: `Z3A2B1`

**풀이**: Z 3번, A 2번, B 1번. 단독 문자라도 `1`이 붙음에 주의.

## 2부. 카이사르 암호화 단독 예측

### 문제 2-1
다음 코드의 출력은?
```python
print(caesar("HELLO", 3))
```

**정답**: `KHOOR`

**풀이**: 각 글자를 알파벳에서 3칸 뒤로 밀기.
- H(+3)=K, E(+3)=H, L(+3)=O, L(+3)=O, O(+3)=R

---

### 문제 2-2
다음 코드의 출력은?
```python
print(caesar("XYZ", 5))
```

**정답**: `CDE`

**풀이**: 알파벳 끝을 넘어가면 처음(A)으로 순환.
- X(+5): X(23)+5=28, 28%26=2 -> C
- Y(+5): Y(24)+5=29, 29%26=3 -> D
- Z(+5): Z(25)+5=30, 30%26=4 -> E

## 3부. 결합 문제

### 문제 3-1: 출력 예측
```python
s = "AAABBC"
result = caesar(rle(s), 5)
print(result)
```

**정답**: `F3G2H1`

**풀이 (단계별)**:
1. `rle("AAABBC")` = `"A3B2C1"`
2. `caesar("A3B2C1", 5)`:
   - 영문자만 시프트, 숫자는 그대로
   - A(+5)=F, 3=3, B(+5)=G, 2=2, C(+5)=H, 1=1
   - 결과: `"F3G2H1"`

---

### 문제 3-2: 실행 속도 비교

다음 두 표현 중 일반적으로 어느 쪽이 더 빠를지 추측해보자.
```python
a = caesar(rle(s), 5)   # 런 길이 부호화 -> 카이사르
b = rle(caesar(s, 5))   # 카이사르 -> 런 길이 부호화
```

**정답**: 일반적으로 **`a` (런 길이 -> 카이사르)** 가 더 빠르다.

**풀이**:

핵심 관찰: 카이사르는 같은 문자를 항상 같은 문자로 바꾼다. 따라서 연속된 같은 문자(예: `AAA`)는 카이사르 후에도 연속 유지(예: `FFF`). 결국 `caesar` 처리 전후의 rle 결과 길이는 같다.

| 항목 | `a = caesar(rle(s), 5)` | `b = rle(caesar(s, 5))` |
|---|---|---|
| 첫 함수 입력 길이 | `len(s)` | `len(s)` |
| 둘째 함수 입력 길이 | `len(rle(s))` (보통 더 짧음) | `len(s)` |
| 총 작업량 | `len(s) + len(rle(s))` | `len(s) + len(s)` |

`rle(s)`가 원본보다 짧으면(연속 반복이 많을 때) `a`가 명백히 빠름.
반대로 모든 문자가 다 다른 데이터면 `rle(s)`가 원본보다 길어져 `b`가 빠를 수도 있음. 일반적인 압축 대상 데이터(반복이 있는 경우)는 `a`가 빠르다.

### 문제 3-3: x 값 찾기
다음 코드를 실행했더니 출력이 `L5I4` 였다. `x`의 값은?
```python
s = "HHHHHEEEE"
result = caesar(rle(s), x)
print(result)   # L5I4
```

**정답**: `x = 4`

**풀이**:
1. `rle("HHHHHEEEE")` = `"H5E4"`
2. `caesar("H5E4", x)` = `"L5I4"`
3. H -> L: H(7) -> L(11), 시프트 4
4. E -> I: E(4) -> I(8), 시프트 4
5. 숫자는 그대로 통과
6. 따라서 `x = 4`

# 허프만 부호화 예제 모음

## 도입

### prefix-free (접두 코드) 성질

허프만 코드는 prefix-free 성질을 가짐.
> 어떤 문자의 코드도 다른 문자 코드의 앞부분이 되지 않는다.

예시 (잘못된 코드): A=0, B=01 -> 비트열 "01"이 'A 다음 ???'인지 'B'인지 모호.
허프만은 모든 문자가 잎(leaf) 노드에만 붙어있어서 자연스럽게 이 성질이 보장됨.
덕분에 비트 스트림에 구분자 없이도 정확히 복원 가능.

### 기본 규칙

1. 빈도가 가장 작은 두 노드를 합쳐 부모 노드 생성 (반복)
2. **빈도가 동률일 때는 알파벳 빠른 노드를 먼저 선택**
3. **합칠 때 좌우 배치는 항상 알파벳/ASCII 순서**:
   - 알파벳 빠른 노드(또는 그룹 내 가장 빠른 문자) -> 0 가지(왼쪽)
   - 알파벳 늦은 노드 -> 1 가지(오른쪽)
4. 새 부모 노드 빈도 = 두 자식 빈도 합
5. 노드가 하나(루트) 남을 때까지 반복

ASCII 순서 참고: 숫자 `'0'`(48) < `'A'`(65) < `'B'`(66) < ... < `'Z'`(90) < `'a'`(97) < ...

### 트리 표기 방향

표준 트리 표기법 (루트가 위, 잎이 아래).

## 예제 1: 합치는 순서가 자명한 경우

**원본 문자열**: `AAADDB0000` (10글자)
**빈도**: 0:4, A:3, D:2, B:1

### 합치는 과정

| 단계 | 후보 (빈도 오름차순) | 동작 (0가지 / 1가지) |
|---|---|---|
| 시작 | B(1), D(2), A(3), 0(4) | - |
| 1 | B(1), D(2), A(3), 0(4) | B + D 합치기. B < D 이므로 B=0, D=1. BD(3) 생성 |
| 2 | A(3), BD(3), 0(4) | 동률(A,BD) -> A 먼저(A < B). A + BD 합치기, A=0, BD=1. ABD(6) 생성 |
| 3 | 0(4), ABD(6) | 0 + ABD 합치기. '0' < 'A' 이므로 0=0, ABD=1 |

### 트리

```
              [ROOT(10)]
              /        \
             0          1
            /            \
         [0(4)]        [ABD(6)]
                        /    \
                       0      1
                      /        \
                   [A(3)]    [BD(3)]
                              /    \
                             0      1
                            /        \
                         [B(1)]    [D(2)]
```

### 코드 부여

| 문자 | 코드 | 길이 |
|---|---|---|
| 0 | `0` | 1 비트 |
| A | `10` | 2 비트 |
| B | `110` | 3 비트 |
| D | `111` | 3 비트 |

### 부호화

원본 `AAADDB0000` ->
- A(10) A(10) A(10) D(111) D(111) B(110) 0(0) 0(0) 0(0) 0(0)
- `1010101111111100000` (19 비트)

원본 10 글자 x 8 비트 = 80 비트
부호화 19 비트
약 76% 절감

## 예제 2: 합친 노드가 다시 위로 올라가는 경우

**원본 문자열**: `ABCCCDDDDD` (10글자)
**빈도**: D:5, C:3, A:1, B:1

### 합치는 과정

| 단계 | 후보 (빈도 오름차순) | 동작 (0가지 / 1가지) |
|---|---|---|
| 시작 | A(1), B(1), C(3), D(5) | - |
| 1 | A(1), B(1), C(3), D(5) | 동률(A,B) -> A 먼저. A + B 합치기, A=0, B=1. AB(2) 생성 |
| 2 | AB(2), C(3), D(5) | AB + C 합치기. A < C 이므로 AB=0, C=1. ABC(5) 생성 |
| 3 | ABC(5), D(5) | 동률(ABC,D) -> ABC 먼저(A < D). ABC + D 합치기, ABC=0, D=1 |

**핵심 포인트**: 단계 1에서 합친 AB(2)가 단계 2에서 C(3)와 합쳐져 ABC(5)로 커짐. 단계 3에서 그 ABC(5)가 D(5)와 동률이 되어 깊은 위치에 박힘.

### 트리

```
              [ROOT(10)]
              /        \
             0          1
            /            \
        [ABC(5)]        [D(5)]
         /    \
        0      1
       /        \
    [AB(2)]   [C(3)]
     /    \
    0      1
   /        \
[A(1)]    [B(1)]
```

### 코드 부여

| 문자 | 코드 | 길이 |
|---|---|---|
| D | `1` | 1 비트 |
| C | `01` | 2 비트 |
| A | `000` | 3 비트 |
| B | `001` | 3 비트 |

### 부호화

원본 `ABCCCDDDDD` ->
- A(000) B(001) C(01) C(01) C(01) D(1) D(1) D(1) D(1) D(1)
- `00000101010111111` (17 비트)

원본 10 글자 x 8 비트 = 80 비트
부호화 17 비트
약 79% 절감

## 예제 3: 빈도가 동률인 노드가 섞인 경우

**원본 문자열**: `AABBCCCDDD` (10글자)
**빈도**: C:3, D:3, A:2, B:2

### 합치는 과정

| 단계 | 후보 (빈도 오름차순) | 동작 (0가지 / 1가지) |
|---|---|---|
| 시작 | A(2), B(2), C(3), D(3) | - |
| 1 | A(2), B(2), C(3), D(3) | 동률(A,B) -> A 먼저. A + B 합치기, A=0, B=1. AB(4) 생성 |
| 2 | C(3), D(3), AB(4) | 동률(C,D) -> C 먼저. C + D 합치기, C=0, D=1. CD(6) 생성 |
| 3 | AB(4), CD(6) | AB + CD 합치기. A < C 이므로 AB=0, CD=1 |

### 트리

```
              [ROOT(10)]
              /        \
             0          1
            /            \
        [AB(4)]        [CD(6)]
         /    \         /    \
        0      1       0      1
       /        \     /        \
    [A(2)]   [B(2)] [C(3)]  [D(3)]
```

### 코드 부여

| 문자 | 코드 | 길이 |
|---|---|---|
| A | `00` | 2 비트 |
| B | `01` | 2 비트 |
| C | `10` | 2 비트 |
| D | `11` | 2 비트 |

모든 코드 길이가 같음 (균형 트리). 압축 효과가 거의 없음.

### 부호화

원본 `AABBCCCDDD` ->
- A(00) A(00) B(01) B(01) C(10) C(10) C(10) D(11) D(11) D(11)
- `00000101101010111111` (20 비트)

원본 10 글자 x 8 비트 = 80 비트
부호화 20 비트
75% 절감 (다른 예제보다 적음)

**교훈**: 빈도가 균등하면 허프만 압축 효과가 떨어짐. 빈도 편차가 클수록 효과적.

## 예제 4: 복호화 (예제 1의 트리 사용)

**받은 비트열**: `10111110`
**사용 트리**: 예제 1의 트리 (0=0, A=10, B=110, D=111)

### 디코딩 과정

루트에서 시작, 비트를 따라가다 잎 도달하면 문자 출력 후 다시 루트로.

| 위치 | 읽은 비트 | 현재 누적 | 트리 위치 | 동작 |
|---|---|---|---|---|
| 1 | 1 | 1 | 1 가지로 | 중간 노드(ABD) |
| 2 | 0 | 10 | 1-0 | **잎 A**, 출력 후 루트로 |
| 3 | 1 | 1 | 1 가지로 | 중간 노드(ABD) |
| 4 | 1 | 11 | 1-1 | 중간 노드(BD) |
| 5 | 1 | 111 | 1-1-1 | **잎 D**, 출력 후 루트로 |
| 6 | 1 | 1 | 1 가지로 | 중간 노드(ABD) |
| 7 | 1 | 11 | 1-1 | 중간 노드(BD) |
| 8 | 0 | 110 | 1-1-0 | **잎 B**, 출력 후 루트로 |

### 복호화 결과: `ADB`

## 예제 5: 복호화 (예제 2의 트리 사용)

**받은 비트열**: `101000001`
**사용 트리**: 예제 2의 트리 (D=1, C=01, A=000, B=001)

### 디코딩 과정

| 위치 | 읽은 비트 | 현재 누적 | 트리 위치 | 동작 |
|---|---|---|---|---|
| 1 | 1 | 1 | 1 가지로 | **잎 D**, 출력 후 루트로 |
| 2 | 0 | 0 | 0 가지로 | 중간 노드(ABC) |
| 3 | 1 | 01 | 0-1 | **잎 C**, 출력 후 루트로 |
| 4 | 0 | 0 | 0 가지로 | 중간 노드(ABC) |
| 5 | 0 | 00 | 0-0 | 중간 노드(AB) |
| 6 | 0 | 000 | 0-0-0 | **잎 A**, 출력 후 루트로 |
| 7 | 0 | 0 | 0 가지로 | 중간 노드(ABC) |
| 8 | 0 | 00 | 0-0 | 중간 노드(AB) |
| 9 | 1 | 001 | 0-0-1 | **잎 B**, 출력 후 루트로 |

### 복호화 결과: `DCAB`

**관찰**: prefix-free 덕분에 구분자 없이도 어디서 한 문자가 끝나고 다음 문자가 시작하는지 자연스럽게 판단 가능.

# 26-2 정보수업 내용 보강 (260516)

## 1. 데이터 압축 - 분류 보충

### GIF는 정말 무손실인가?

엄밀히 말하면 **조건부 무손실**.

- **압축 알고리즘(LZW) 자체는 무손실**: 픽셀 데이터를 그대로 복원함
- **단, GIF는 색상을 256색(8비트 팔레트)으로 제한**: 풀컬러(24비트, 약 1670만색) 이미지를 GIF로 저장하면 색상 양자화 단계에서 손실 발생

| 입력 이미지 | GIF 저장 결과 |
|---|---|
| 256색 이하 (아이콘, 단순 그래픽, 로고) | 완전 무손실 |
| 풀컬러 사진 | 색상 손실 (압축은 무손실이지만 팔레트 변환에서 손실) |

**교육 현장에서의 처리**:
- 교과서/내신 출제 기준으로는 "GIF = 무손실"로 단순화해서 가르침
- PNG와 묶어 무손실 진영으로 분류하는 게 일반적
- 학생 답안 기준은 "무손실"로 통일
- 다만 교사 차원에서는 "팔레트 색상 제한이 있는 무손실"이라는 뉘앙스를 알고 있을 것

## 2. 데이터 단위 - KB와 KiB 문제

### 기비바이트(GiB)란?

이진 접두어(binary prefix). 컴퓨터에서 단위 표기가 두 가지로 갈리는 문제에서 등장.

| 십진 (SI 단위) | 값 | 이진 (IEC 단위) | 값 |
|---|---|---|---|
| 1 KB (킬로바이트) | 1000 byte | 1 KiB (키비바이트) | 1024 byte |
| 1 MB (메가바이트) | 1000 KB | 1 MiB (메비바이트) | 1024 KiB |
| 1 GB (기가바이트) | 1000 MB | 1 GiB (기비바이트) | 1024 MiB |
| 1 TB (테라바이트) | 1000 GB | 1 TiB (테비바이트) | 1024 GiB |

### 왜 두 개가 있나?
- 원래 K/M/G/T는 SI 단위로 **1000 배수**가 정확한 정의
- 하지만 컴퓨터는 2진법 기반이라 **1024 배수**가 자연스러움 (2^10 = 1024)
- 관습적으로 KB/MB/GB를 1024 배수로 써왔음 -> 혼란 발생
- IEC가 1998년에 KiB/MiB/GiB/TiB(이진 접두어)를 새로 정의해서 구분

### 실제로 보이는 혼란
- **하드디스크 제조사**: "1TB" = 1,000,000,000,000 byte (SI, 1000 기반)
- **Windows**: "GB"로 표시하지만 내부 계산은 1024 기반 -> 1TB 디스크가 931GB로 보임
- **macOS/Linux**: GB와 GiB를 점점 구분해서 표기

### 그림1과의 관계
손글씨 노트에는 `1 KB = 1024 byte`로 적혀있음. 엄밀히는 `1 KiB = 1024 byte`가 맞지만, 교과서/내신에서는 대부분 "1KB = 1024 byte"로 가르치므로 일반적으로는 그대로 외워둠.

### 학생이 시험에서 어떻게 답해야 하나?
**선생님께 직접 확인 권장**. 학교/교과서/출제자에 따라 기준이 다르므로:
- 수업 시간에 1KB = 1024 byte로 가르쳤다면 그대로 답
- 단, 만약 시험에 "기비바이트(GiB)" 같은 단위가 보기로 등장한다면 출제 의도를 다르게 잡아야 함
- 출제 범위와 표기 기준을 선생님께 명확히 여쭤보고 정리할 것

### 실제 현장은 더 혼란스러움 (1024 vs 1000 혼용)

"컴퓨터 용량은 1024 기준"이라는 통념은 부분적으로만 맞음. 분야마다 기준이 다름.

| 영역 | 기준 | 예시 |
|---|---|---|
| 저장장치 제조사 (HDD/SSD/USB) | SI (1000) | "1TB SSD" = 1,000,000,000,000 byte |
| RAM (메모리) | 이진 (1024) | "16GB RAM" = 실제로는 16 GiB = 17,179,869,184 byte |
| Windows 탐색기 표시 | 이진 (1024)이지만 단위 표기는 GB/TB | "1TB 디스크"가 약 931GB로 보임 |
| macOS/Linux 최근 표기 | SI (1000), GiB는 별도 구분 | 1TB 디스크를 1TB로 표시 |
| 네트워크 속도 | SI (1000) | 1 Gbps = 10^9 bps |

**왜 RAM은 1024 기반인가**
하드웨어 구조상 메모리는 주소선 개수의 2의 거듭제곱으로 만들어짐. 주소선 30개 -> 2^30 byte = 1 GiB. RAM은 본질적으로 이진 단위로만 만들어짐.

**왜 하드디스크는 1000 기반인가**
플래터/셀 단위가 2의 거듭제곱일 필요가 없음. 마케팅상 큰 숫자 쓰려고 SI 표기 선택 (1TB가 0.91TiB보다 보기 좋음).

**Windows의 혼란**
- Windows는 1024 기반으로 계산하면서 단위는 GB/TB로 표기
- 1TB 디스크(제조사 표기: 10^12 byte)를 꽂으면 Windows에서 약 0.91 TB로 보임
- 정확히는 Windows가 "TiB"로 표기했어야 맞음
- 사용자가 "내 1TB 어디 갔지?" 하게 되는 원인

**정리**
- RAM: 1024 기반
- 저장장치: 1000 기반
- OS 표시: 둘이 섞여있어 헷갈림
- IEC가 KiB/MiB/GiB를 만든 이유가 바로 이 혼란을 정리하려던 것
- 현장 표기가 아직 안 따라가서 혼란이 남아있는 상태

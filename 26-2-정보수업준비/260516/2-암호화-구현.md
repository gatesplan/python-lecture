## 1. 런 길이 부호화 (Run-Length Encoding)

연속된 같은 문자를 `문자 + 개수` 로 표현.

```python
def rle_encode(s):
    result = ""
    count = 1
    for i in range(len(s)):
        # 마지막 글자이거나, 다음 글자가 다르면 출력
        if i == len(s) - 1 or s[i] != s[i+1]:
            result += s[i] + str(count)
            count = 1
        else:
            count += 1
    return result


def rle_decode(s):
    result = ""
    for i in range(0, len(s), 2):
        ch = s[i]
        num = int(s[i+1])
        result += ch * num
    return result


# 사용 예
text = "AAADDB0000"
encoded = rle_encode(text)
print(encoded)              # A3D2B104
print(rle_decode(encoded))  # AAADDB0000
```

**핵심 동작**:
- `for i in range(len(s))`로 한 글자씩 순회
- 다음 글자(`s[i+1]`)가 같으면 `count` 증가, 다르면(또는 마지막이면) `문자+count` 출력하고 리셋
- 디코더는 두 글자씩 끊어 읽음 (문자, 숫자 한 자리 가정)

**주의/한계**:
- 위 코드는 **count가 한 자리 숫자(1~9)** 일 때만 정상 동작. 같은 문자가 10번 이상 연속되면 디코더가 잘못 해석함
- 데이터에 숫자가 섞여 있으면 `A3`이 'A 3번'인지 'A 다음 3'인지 구분 불가. 실제 RLE는 문자만 다루거나, 특수 구분자/이진 형식을 씀

## 2. 허프만 부호화 (Huffman Coding)

빈도 기반 가변 길이 코드. 빈도 높은 문자에 짧은 코드, 낮은 문자에 긴 코드.

두 가지 구현 방식 제시:
- **2-1. 표준 구현**: 트리 클래스 + 우선순위 큐(힙). 알고리즘 책/현업 표준 패턴
- **2-2. 학생친화 구현**: 트리 클래스/힙 없이 리스트와 사전만으로 손풀이 흐름 그대로

### 2-1. 표준 구현 (Node 클래스 + heapq)

```python
import heapq
from collections import Counter


class Node:
    def __init__(self, freq, key, char=None, left=None, right=None):
        self.freq = freq      # 빈도(이 노드/하위 트리의 총합)
        self.key = key        # 정렬 키(그룹 내 알파벳 빠른 문자)
        self.char = char      # 잎 노드만 문자 보유, 내부 노드는 None
        self.left = left
        self.right = right

    def __lt__(self, other):
        # heapq가 노드끼리 비교할 때 호출하는 메서드
        # 빈도 오름차순, 동률이면 알파벳 빠른 순
        return (self.freq, self.key) < (other.freq, other.key)


def huffman_build_tree(text):
    # 1) 빈도 계산
    freq = Counter(text)

    # 2) 각 문자에 대해 잎 노드를 만들고 최소 힙으로 변환
    heap = [Node(f, c, char=c) for c, f in freq.items()]
    heapq.heapify(heap)

    # 3) 노드 하나(루트) 남을 때까지 합치기 반복
    while len(heap) > 1:
        first = heapq.heappop(heap)   # 가장 작은 노드
        second = heapq.heappop(heap)  # 두 번째로 작은 노드

        # 좌우 배치
        # - 빈도 다르면: 큰 빈도가 왼쪽(0), 작은 빈도가 오른쪽(1)
        # - 빈도 동률: first가 알파벳 빠르므로 왼쪽(0), second가 오른쪽(1)
        if first.freq == second.freq:
            left_child, right_child = first, second
        else:
            left_child, right_child = second, first

        merged = Node(
            freq=first.freq + second.freq,
            key=min(first.key, second.key),
            left=left_child,
            right=right_child
        )
        heapq.heappush(heap, merged)

    return heap[0]


def huffman_codes(root, prefix="", codes=None):
    # 트리 재귀 순회로 각 잎의 코드 추출
    if codes is None:
        codes = {}
    if root.char is not None:
        codes[root.char] = prefix if prefix else "0"
    else:
        if root.left:
            huffman_codes(root.left, prefix + "0", codes)
        if root.right:
            huffman_codes(root.right, prefix + "1", codes)
    return codes


def huffman_encode(text):
    root = huffman_build_tree(text)
    codes = huffman_codes(root)
    encoded = "".join(codes[c] for c in text)
    return encoded, root, codes


def huffman_decode(encoded, root):
    # 비트 따라 트리를 내려가다 잎에 도달하면 문자 출력, 루트로 복귀
    result = []
    node = root
    for bit in encoded:
        node = node.left if bit == "0" else node.right
        if node.char is not None:
            result.append(node.char)
            node = root
    return "".join(result)


# 사용 예
text = "AAADDB0000"
encoded, root, codes = huffman_encode(text)
print(codes)                            # {'A': '00', 'D': '010', 'B': '011', '0': '1'}
print(encoded)                          # '0000000100100111111'
print(huffman_decode(encoded, root))    # AAADDB0000
```

#### 코드 요소 설명

| 요소 | 역할 |
|---|---|
| `class Node` | 트리의 한 노드. 잎이면 `char` 보유, 내부 노드면 `left`/`right` 자식 보유 |
| `__lt__` | 파이썬 비교 연산자 `<` 오버라이드. `heapq`가 노드끼리 비교할 때 자동 호출 |
| `Counter` | 문자열의 문자별 빈도 자동 계산 (`collections` 모듈) |
| `heapq` | 최소 힙 모듈. `heappush`/`heappop`으로 항상 최소 원소를 O(log n)에 꺼냄 |
| 재귀 (`huffman_codes`) | 트리 잎까지 내려가며 누적된 0/1 경로를 그 문자의 코드로 저장 |

#### 실행 추적 (입력: `"AAADDB0000"`)

**1단계: 빈도 계산**
```
freq = {'A': 3, 'D': 2, 'B': 1, '0': 4}
```

**2단계: 잎 노드 4개로 힙 초기화** (heapify 후 `(freq, key)` 오름차순)
```
heap: [Node(1,'B',char='B'), Node(2,'D',char='D'),
       Node(3,'A',char='A'), Node(4,'0',char='0')]
```

**3단계: 루프 1회차**
```
first  = heappop -> Node(1,'B',char='B')   # 빈도 최소
second = heappop -> Node(2,'D',char='D')

freq 다름 (1 != 2):
  큰 쪽(D) -> 왼쪽(0), 작은 쪽(B) -> 오른쪽(1)
merged = Node(3, 'B', left=D_node, right=B_node)
heappush(merged)

heap: [Node(3,'A',...), Node(3,'B',merged), Node(4,'0',...)]
```

**4단계: 루프 2회차**
```
first  = heappop -> Node(3,'A',char='A')      # (3,'A') < (3,'B')
second = heappop -> Node(3,'B',merged)

freq 동률:
  알파벳 빠른 first(A) -> 왼쪽(0), second(BD 그룹) -> 오른쪽(1)
new_merged = Node(6, 'A', left=A_node, right=merged)
heappush(new_merged)

heap: [Node(4,'0',...), Node(6,'A',new_merged)]
```

**5단계: 루프 3회차**
```
first  = heappop -> Node(4,'0',char='0')
second = heappop -> Node(6,'A',new_merged)

freq 다름 (4 != 6):
  큰 쪽(ABD 그룹) -> 왼쪽(0), 작은 쪽(0) -> 오른쪽(1)
root = Node(10, '0', left=new_merged, right=0_node)

heap: [root]  -> 루프 종료
```

**6단계: `huffman_codes(root, "")` 재귀 호출**
```
root (char=None, prefix="")
 -> huffman_codes(root.left=new_merged, "0")
      new_merged (char=None)
       -> huffman_codes(A_node, "00")
            char='A' -> codes['A'] = '00'
       -> huffman_codes(merged, "01")
            merged (char=None)
             -> huffman_codes(D_node, "010")
                  char='D' -> codes['D'] = '010'
             -> huffman_codes(B_node, "011")
                  char='B' -> codes['B'] = '011'
 -> huffman_codes(root.right=0_node, "1")
      char='0' -> codes['0'] = '1'

최종 codes = {'A': '00', 'D': '010', 'B': '011', '0': '1'}
```

**7단계: 부호화**
```
text  = "AAADDB0000"
codes = {'A': '00', 'D': '010', 'B': '011', '0': '1'}
encoded = '00' + '00' + '00' + '010' + '010' + '011' + '1'+'1'+'1'+'1'
        = '0000000100100111111'  (19 비트)
```

**시간복잡도**: 힙 연산이 O(log n) x n번 = O(n log n). 단 n은 문자 종류 수.

### 2-2. 학생친화 구현 (리스트 + 사전)

손으로 푸는 순서를 그대로 코드로 옮긴 방식. 트리 클래스/힙 없이 그룹 리스트만 정렬하며 합치고, 코드 사전을 직접 갱신.

```python
from collections import Counter


def huffman_encode(s):
    # 1. 빈도 계산
    freq = Counter(s)

    # 2. 각 그룹을 [빈도, 대표키, 멤버 문자 리스트] 형태로 만들기
    groups = []
    for ch, f in freq.items():
        groups.append([f, ch, [ch]])

    # 3. 각 문자의 코드를 빈 문자열로 초기화
    codes = {}
    for ch in freq:
        codes[ch] = ''

    # 4. 그룹 하나 남을 때까지 합치기 반복
    while len(groups) > 1:
        # 빈도 오름차순, 동률이면 알파벳 빠른 순으로 정렬
        groups.sort(key=lambda g: (g[0], g[1]))

        # 가장 작은 두 그룹 꺼내기
        first = groups.pop(0)
        second = groups.pop(0)

        # 좌우 배치 결정
        # - 빈도 다르면: 큰 빈도가 0, 작은 빈도가 1
        # - 빈도 동률이면: 알파벳 빠른 게 0, 늦은 게 1
        if first[0] == second[0]:
            # 동률: first가 알파벳 빠르므로 0
            zero_members = first[2]
            one_members = second[2]
        else:
            # 동률 아님: first가 작은 빈도이므로 1, second가 큰 빈도이므로 0
            zero_members = second[2]
            one_members = first[2]

        # 각 멤버의 기존 코드 앞에 0 또는 1을 붙임
        for ch in zero_members:
            codes[ch] = '0' + codes[ch]
        for ch in one_members:
            codes[ch] = '1' + codes[ch]

        # 합친 새 그룹을 리스트에 다시 넣기
        merged = [
            first[0] + second[0],
            min(first[1], second[1]),  # 그룹 대표키는 멤버 중 알파벳 빠른 것
            first[2] + second[2]
        ]
        groups.append(merged)

    # 5. 코드 사전대로 부호화
    encoded = ''
    for ch in s:
        encoded += codes[ch]
    return encoded, codes


def huffman_decode(encoded, codes):
    # 코드 -> 문자 역매핑
    reverse = {}
    for ch, code in codes.items():
        reverse[code] = ch

    # prefix-free 성질 덕분에, 누적 비트열이 어떤 코드와 일치하는 순간 그 글자
    result = ''
    buffer = ''
    for bit in encoded:
        buffer += bit
        if buffer in reverse:
            result += reverse[buffer]
            buffer = ''
    return result


# 사용 예
text = "AAADDB0000"
encoded, codes = huffman_encode(text)
print(codes)                            # {'A': '00', 'D': '010', 'B': '011', '0': '1'}
print(encoded)                          # '0000000100100111111'
print(huffman_decode(encoded, codes))   # AAADDB0000
```

**핵심 동작**:
- `Counter`로 빈도 계산
- 각 그룹은 `[빈도, 대표키, 멤버들]` 리스트 -> 정렬해서 가장 작은 두 개 꺼냄
- 합칠 때 한쪽 그룹 멤버 코드 앞에 '0', 다른 쪽에는 '1' 붙이기
- 새 합친 그룹을 다시 리스트에 추가, 반복
- 복호화는 누적 buffer가 어떤 코드와 일치하면 그게 정답 (prefix-free 덕분)

## 3. 카이사르 암호화 (Caesar Cipher)

알파벳을 일정 칸만큼 밀어서 치환.

```python
def caesar_encode(text, shift):
    result = []
    for ch in text:
        if 'A' <= ch <= 'Z':
            result.append(chr((ord(ch) - ord('A') + shift) % 26 + ord('A')))
        elif 'a' <= ch <= 'z':
            result.append(chr((ord(ch) - ord('a') + shift) % 26 + ord('a')))
        else:
            result.append(ch)  # 알파벳이 아닌 문자는 그대로
    return "".join(result)


def caesar_decode(text, shift):
    return caesar_encode(text, -shift)


# 사용 예
text = "abcd"
encoded = caesar_encode(text, 4)
print(encoded)                       # efgh
print(caesar_decode(encoded, 4))     # abcd

# 알파벳 끝을 넘어가는 경우 (% 26 으로 순환)
print(caesar_encode("xyz", 3))       # abc
```

**핵심 동작**:
- `ord()` 로 문자 -> 숫자 변환
- `(원래 위치 + shift) % 26` 으로 순환 처리
- `chr()` 로 다시 문자로 변환

# 전치 암호 (Transposition Cipher) - 출제 유형 정리

학교 수업에서 다룬 "ABCDEFG를 132 키로 섞기" 내용. 글자를 바꾸지 않고 순서만
재배열하는 방식이라 전치 암호(순열 암호, Permutation Cipher)라 부른다.
글자 자체를 바꾸는 시저 암호(치환, 260516 폴더)와 짝을 이룬다.

## 동작 원리

키 "132" = 블록 안 글자를 1번, 3번, 2번 순서로 재배열.
0-indexed로는 [0, 2, 1], 블록 크기 k = 3.

```
ABCDEFG 를 3글자씩 끊어서
[ABC][DEF][G]
 -> 각 블록을 0,2,1 순서로
[ACB][DFE][G]
= ACBDFEG
```

## 채택 구현: 방식 2 (이중 반복문, 인덱스 식)

세 가지 구현 방식(교란인덱스 리스트 / 이중 반복문 / 단일 for문 압축)을 검토한 결과,
출제 베이스는 방식 2로 단일 채택한다.
이유: 부분 블록(길이가 k 배수가 아닐 때)을 내부 가드로 가장 자연스럽게 처리.
방식 1, 3은 부분 블록에서 인덱스가 어긋나 학생/채점 모두 부담.

```python
result = ''
for b in range(0, len(s), k):   # 블록 시작 위치
    for j in key:               # 블록 내 키 순서 (key = [0, 2, 1])
        if b + j < len(s):      # 부분 블록 처리 가드
            result += s[b + j]
```

## 출제 가능 유형

| 유형 | 비우는/묻는 곳 | 난이도 |
|---|---|---|
| 결과 추적 | 코드 주고 "ABCDEFG -> ?" 손으로 추적 | 하 |
| 빈칸: 핵심 인덱스 | result += s[b + __] | 하~중 |
| 빈칸: 루프 범위 | range(0, len(s), __) 또는 for j in __ | 중 |
| 빈칸: 부분블록 가드 | if b + j < ___: | 중 |
| 코드 작성 | 키와 문자열 주고 함수 전체 작성 | 중~상 |

## 출제 시 결정할 사항

1. 부분 블록 포함 여부
   - 길이를 k 배수로 맞추면(예: ABCDEF) 가드가 무의미해져 쉬워짐.
   - 안 맞추면(예: ABCDEFG) 마지막 블록 가드가 변별 포인트가 됨.
   - 난이도에 따라 선택.

2. 키 표현 방식
   - "132"(1-indexed 문자열)를 코드에서 [0, 2, 1]로 변환해 쓰게 할지,
   - 아예 key = [0, 2, 1]로 줄지.
   - 변환까지 시키면 한 단계 어려워짐.

## 복호화 문제 주의

키 "132"는 역순열이 자기 자신이다(2, 3번 위치만 스왑).
복호화 문제로 쓰면 암호화와 동일해져 변별이 안 됨.
복호화를 출제하려면 역순열이 다른 키를 골라야 한다.
예: "231"(역순열 "312").

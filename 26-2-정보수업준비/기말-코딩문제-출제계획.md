# 기말고사 코딩 문제 출제 계획

기말 범위(자료구조/클래스/데이터/AI) 코딩 문제의 큰 갈래와 갈래별 소재 풀 정리.
출처: 기말고사 공지(기말고사-시험범위.md), 260530/260613 수업 노트북, 학생 이론 요약.

## 공통 형식 (전 갈래 적용)

- 코드는 **주어지고**, 학생은 읽고 답한다. 특정 자료구조 "전용" 문제는 없고, 자료구조/개념은 코드 흐름 속에서 쓰인다.
- 문제 유형: **실행결과 구하기 / 중간단계 값 추적 / 동작 설명(주관식) / 가벼운 빈칸채우기**
- **코드 전체 작성형은 없음** (모든 갈래 공통, 클래스 포함)
- 갈래는 "문제 형식"이 아니라 **코드에 등장하는 개념**으로 나뉜다.

## 배운 범위 제약 (코드 작성 시 반드시 준수)

- 딕셔너리 메서드는 get() / items() / values() 만 사용 (setdefault, keys(), dict() 등 금지)
- 키+값 순회는 반드시 for k, v in d.items() (for k, v in d 는 동작 안 함)
- enumerate 미사용, 리스트 컴프리헨션 미사용 (정식으로 안 배움)
- zip 사용 가능 (두 리스트 동시 순회는 배움)
- join 미사용 (문자열 연결은 += 로)
- 정렬 알고리즘(버블/삽입/선택) 소재 제외, 리스트 .sort() 도 지양

## 갈래 1: 자료구조 + 제어문

- 형식: 4종 모두 사용 (실행결과/추적/빈칸/설명)
- 구성: "자료구조 + 특정주제"(수론/암호화/진법/회문 등 명시 주제 결합)와 "자료구조 + 자유주제"(임의 창작 흐름)를 섞어 다양하게. 난이도는 단순 추적보다 한 단계 위(개념 결합, 흐름 길이 증가).
- 소재 풀:
  - 수론: 최대공약수/최소공배수, 소수 판별, 약수/배수
  - 수열/집계: 팩토리얼, 피보나치, 누적합, 최대/최소/평균
  - 자료구조 가공(중간 난이도 기반): 슬라이싱/스왑/복사 함정, 2차원(행/열/대각), 딕셔너리 카운팅/순회, 슬라이딩 윈도우
  - 자릿수/회문 처리, 탐색(선형/이진), 별/패턴 출력, 진법 변환
  - 암호화(260516 자료 기반): 전치/카이사르 (코드 등장 가능), 허프만 부호화 (코드 없이 변환 수행 요구 - 공통 형식 예외)
  - 암호화 조합/연쇄(변환 수행형): 카이사르 -> 허프만 등 2~3가지 변환을 순서대로 적용한 뒤 최종 결과를 묻기 (코드 없이 손으로 수행, 규칙은 condition-box로 제시, 공통 형식 예외)
  - 용량 단위: bit/byte/KB/MB/GB 등 단위 변환 (이론 단독 가능, 1024 배수 변환을 코드와 결합 가능)
  - 서식 지정자(출력 정렬): 소수점 자릿수(:.2f), 정렬/폭(:>N 우, :<N 좌, :^N 가운데), 조합(:>7.2f). "표를 print로 깔끔하게 출력"(공지 강조) 대응
  - 창작 제어문+자료구조 조합
- 주의: 이진탐색/진법변환은 실제로 안 배웠으면 제외

## 갈래 2: 클래스/객체

- 형식: 공통 (실행결과/추적/설명/빈칸, 코드 작성형 없음)
- 코드 등장 범위:
  - 단일 클래스 정의
  - __init__으로 속성 초기화 (기본값 매개변수 포함)
  - 메서드 정의/호출
  - 메서드로 객체 내부값(self.x) 변경 -> 상태 변화 "중간단계 추적"에 활용
  - 제외: 상속, 클래스 변수, 복잡한 객체 간 상호작용

## 갈래 3: 데이터 처리 (pandas)

- 형식: 빈칸채우기 + 동작 설명 (실행결과/추적은 부적합)
- 소재 풀:
  - CSV 입출력: read_csv (인코딩 예외처리 try utf-8 / except cp949), to_csv
  - 기초 탐색: df.shape, df.head(), df.info(), df.describe()
  - 결측치 처리: df.isnull().sum(), df.dropna(), 평균값으로 fillna
  - 컬럼 선택/파생: df[컬럼리스트].copy(), 리스트 컴프리헨션, sum(axis=1)
- 근거: 260530/note2.ipynb

## 갈래 4: AI 흐름 (sklearn)

- 형식: 빈칸채우기 + 동작 설명 + 결과 해석 서술형
- 핵심: 데이터 로드 -> 분리 -> 학습 -> 예측/평가 흐름
- 소재 풀:
  - 공통 흐름 골격: train_test_split(80/20), model.fit(X_train, y_train), model.predict(X_test)
    - 빈칸 예: train_test_split(X, y, test_size=[[]], random_state=42) 등 파라미터 빈칸
  - 회귀(선형): LinearRegression, r2_score(R^2), mean_absolute_error(MAE)
  - 분류: k-NN, 의사결정트리 등 + 정확도/혼동행렬
  - 군집(k-Means): KMeans(n_clusters), labels_, inertia_(Elbow)
  - 결과 해석 서술형: 데이터와 선형회귀 결과 그래프를 주고 결정계수(R^2)가 낮은 이유를 분석해 서술 (예: 비선형 관계, 큰 분산/산포, 이상치, 약한 상관)
- 근거: 260613 회귀/군집 노트북, 기계학습-이론요약.md
- 미해결(이미지): 결과 그래프 제시 필요. image 블록 타입은 있으나 ProblemRenderer의 img에 src 미연결로 현재 표시 불가 -> 렌더러 수정 + 이미지 저장 위치(public/base64) 결정 필요

# 버블소트 문제 기획

## 대상 코드

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-1-i):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```

| 주제 | 상태 | 비고 |

| 빈칸채우기 | 상세 (본 문서) | 난이도 4티어 |
| 결과 예측 | 초안 미작성 | 입력값별 최종/중간 상태 |
| 변형/확장 | 초안 미작성 | 내림차순, 조기종료(swapped), 2D 정렬 등 |
| 디버깅 | 초안 미작성 | off-by-one, 비교방향 반전, 인덱스 오류 |

## 빈칸채우기 상세

### 블랭크 후보 식별

| ID | 위치 | 블랭크 대상 | 개념 포인트 |
|----|------|-------------|-------------|
| A | `n = len(arr)` | `len(arr)` | 길이 구하기 |
| B | `for i in range(n)` | `n` | 외부 루프 횟수 |
| C | `for j in range(0, n-1-i)` | `n-1-i` | 왜 마지막 원소 건너뛰는가 (-i 축소) |
| D | `if arr[j] > arr[j+1]` | 비교연산자 `>` | 오름차순 비교 방향 |
| E | `if arr[j] > arr[j+1]` | 인덱스 `j`, `j+1` | 인접 쌍 참조 |
| F | `arr[j], arr[j+1] = arr[j+1], arr[j]` | swap 우변 | swap 관용구 |
| G | `arr[j], arr[j+1] = arr[j+1], arr[j]` | swap 좌변 | swap 좌변 타겟 |
| H | `return arr` | `arr` | 반환값 |

### 난이도 티어 (변형 후보)

#### Tier 1 - 하

- **변형 1** (A): `n = [[]]` — `len()` 호출 회상
- **변형 2** (B): `for i in range([[]]):` — 전체 패스 수 이해

#### Tier 2 - 중

- **변형 3** (D): `if arr[j] [[]] arr[j+1]:` — 오름차순 비교 방향
- **변형 4** (C): `for j in range(0, [[]]):` — `-i` 축소 이해. 변형 3보다 개념 부하가 높음

#### Tier 3 - 상 (swap 관용구)

- **변형 5** (F): `arr[j], arr[j+1] = [[]], [[]]` — swap 우변의 교차 배치
- **변형 6** (G): `[[]], [[]] = arr[j+1], arr[j]` — swap 좌변 타겟 지정

#### Tier 4 - 최상 (다중 상호 종속 / 복합)

- **변형 7** (E): `if arr[[[]]] > arr[[[]]]:` — 인접 쌍 `(j, j+1)` 복원
- **변형 8** (B+C): 외부/내부 루프 범위 동시 블랭크 — 패스 구조 전체 이해

### 미결정 사항

- 각 변형에 `output-sample` 을 붙일지 (빈칸채우기인데 출력 예시까지 요구할지)
- 함수 호출부 `print(bubble_sort([...]))` 를 하단에 붙여 결과 예측과 결합할지
- 힌트(주석) 제공 여부 — 기본은 비서술 변수명 + 힌트 없음
